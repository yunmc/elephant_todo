# 经典皮肤（Classic Skin）设计规范

> 日式简约暖色调 · Elephant Todo 默认皮肤

---

## 1. 设计理念

| 关键词 | 说明 |
|--------|------|
| **日式简约** | 适度对比、低饱和度、留白充足，视觉节奏平缓 |
| **暖色调** | 以米色 / 灰棕 / 藏青 / 金色为基底，营造纸质手帐质感 |
| **克制装饰** | 无多余阴影与圆角，用色彩层级本身传达信息 |
| **通体全暖** | 阴影、边框、语义色全部暖色处理，形成完整色彩气场 |
| **舒适阅读** | 文字色避免纯黑 `#000`，使用灰棕主色 `#5d574f` |

## 5. 组件规范

### 5.1 页面容器 `.page-container`

- 水平内边距：`var(--page-px)` = 16px
- 顶部安全区兼容：`max(var(--page-px), calc(env(safe-area-inset-top) + 8px))`
- 底部预留导航栏空间：`var(--page-px) + var(--bottom-tab-height) + env(safe-area-inset-bottom)`
- 最大宽度：600px，居中
- 最小高度：100vh

### 5.2 页面标题 `.page-title`

- 字号 22px、字重 **500**、字间距 1px
- 下边距 16px

### 5.3 卡片 `.jp-card`

- 背景：`var(--color-bg-card)`
- 圆角：`var(--radius-md)` = 12px
- 内边距：20px
- 阴影：`var(--shadow-md)`
- 无描边

### 5.4 卡片标题 `.jp-card-title`

- 字号 17px、字重 **500**
- 颜色：`var(--color-text)`
- 下边距 15px

### 5.5 信息行 `.jp-info-row` + `.jp-label` + `.jp-value`

- 标签宽度 80px，颜色 `var(--color-text-secondary)`

- 背景：`var(--color-primary)` 藏青

### 5.21 可复用输入行组件 `<JpAddRow>`

封装了「输入框 + 按钮」的统一添加行，通过插槽支持多种变体：

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `v-model` | `string` | `''` | 输入值 |
| `placeholder` | `string` | `'请输入...'` | 占位文字 |
| `btnText` | `string` | `'添加'` | 按钮文字 |
| `maxlength` | `number` | `100` | 最大输入长度 |

| 插槽 | 用途 |
|------|------|
| `#prepend` | 前置元素（如 `.jp-emoji-btn` emoji 选择器） |
| `#append` | 后置元素（如 `.color-input` 颜色选择器） |

**使用示例：**

```vue
<!-- 基础添加行 -->
<JpAddRow v-model="name" placeholder="新分类名称" @submit="handleAdd" />

<!-- 带颜色选择器 -->
<JpAddRow v-model="name" placeholder="新标签" @submit="handleAdd">
  <template #append>
    <input type="color" v-model="tagColor" class="color-input" />
  </template>
</JpAddRow>

<!-- 带 emoji 选择器 -->
<JpAddRow v-model="name" placeholder="添加新习惯" @submit="handleAdd">
  <template #prepend>
    <button class="jp-emoji-btn" @click="showPicker = true">{{ icon }}</button>
  </template>
</JpAddRow>
```

### 5.25 场景豁免规则

以下场景不强制遵循 40px 高度 / `.jp-btn` 规范：

1. **Modal 表单内的 NaiveUI 按钮**：如 n-modal 内的保存/取消按钮，抽屉内的表单按钮
2. **列表内操作按钮**：如删除、编辑等 `size="tiny"` 的文字按钮
3. **导航按钮**：如“去添加习惯”“去逛逛商店”等引导性按钮
4. **CTA 全宽按钮**：如商店购买按钮 `size="large"` block
5. **Admin 后台**：独立配色体系
6. **Period 经期追踪**：粉色特征色豁免

## 6. iOS / Capacitor 适配

- 安全区域通过 `env(safe-area-inset-*)` 动态适配
- `.capacitor-ios`：禁用长按菜单（`-webkit-touch-callout: none`），输入框保留文字选择
- Naive UI 消息/通知容器：顶部偏移 `env(safe-area-inset-top) + 8px` 避免刘海/灵动岛遮挡
- 最小触控区域：44×44px（`button, [role='button'], .touchable`）

## 10. 无障碍 / 焦点态

所有可聚焦元素均添加 `:focus-visible` 样式，确保键盘导航可视：

```css
:focus-visible {
  outline: 2px solid var(--color-primary);
  outline-offset: 2px;
}
```

对比度标准（WCAG AA）：

| 场景 | 要求 | 当前 |
|------|------|------|
| 主文字 on card | ≥4.5:1 | ✅ `#5d574f` on `#fffdf9` ≈ 5.2:1 |
| 次要文字 on card | ≥3:1（大字）| ✅ `#857f75` on `#fffdf9` ≈ 3.5:1 |
| Pill 白字 on active | ≥4.5:1 | ✅ `#fff` on `#7a6e5e` ≈ 4.9:1 |
| Warning on card | ≥3:1 | ✅ `#b8923e` on `#fffdf9` ≈ 3.8:1 |
| Danger on card | ≥3:1 | ✅ `#bf6f65` on `#fffdf9` ≈ 3.6:1 |
| Success on card | ≥3:1 | ✅ `#7a9e6e` on `#fffdf9` ≈ 3.0:1 |

## 11. 开发约定

1. **所有颜色必须走 CSS 变量**，禁止在页面 `<style>` 中直接写 `#hex` / `rgb()` / `hsl()`
2. **新增颜色** → 先在 `_classic.scss` 中定义变量，然后在页面中通过 `var(--color-xxx)` 引用
3. **Naive UI 组件色** → 同步更新 `nuxt.config.ts` 中的 `themeConfig.light` / `themeConfig.dark`
4. **透明度变体** → 使用 `rgba(var(--color-primary-rgb), 0.15)` 模式，而非直接写带透明度的 hex
5. **深色模式** → 每个新变量都需要在 `.dark` 块中定义对应值，不可遗漏
6. **输入行优先用组件** → 新增「输入 + 按钮」场景，优先使用 `<JpAddRow>` 或 `<JpSelectCreate>` 组件，而非手写 HTML
7. **换肤友好** → 所有样式通过 CSS 变量控制，组件样式定义在 `main.scss` 中，不在组件 `<style scoped>` 中硬编码色值
8. **特殊模块豁免**：
   - `pages/admin/*` — 后台独立配色体系，不受经典皮肤约束
   - `pages/period/*` — 经期追踪使用粉色 `#ec4899` 为领域特征色
