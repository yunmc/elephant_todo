# 经典皮肤（Classic Skin）设计规范

> 日式简约暖色调 · Elephant Todo 默认皮肤

---

## 1. 设计理念

| 关键词 | 说明 |
|--------|------|
| **日式简约** | 适度对比、低饱和度、留白充足，视觉节奏平缓 |
| **暖色调** | 以米色 / 灰棕 / 藏青 / 金色为基底，营造纸质手帐质感 |
| **克制装饰** | 无多余阴影与圆角，用色彩层级本身传达信息 |
| **通体全暖** | 阴影、边框、语义色全部暖色处理，形成完整色彩气场 |
| **舒适阅读** | 文字色避免纯黑 `#000`，使用灰棕主色 `#5d574f` |

## 5. 组件规范

### 5.1 页面容器 `.page-container`

- 水平内边距：`var(--page-px)` = 16px
- 顶部安全区兼容：`max(var(--page-px), calc(env(safe-area-inset-top) + 8px))`
- 底部预留导航栏空间：`var(--page-px) + var(--bottom-tab-height) + env(safe-area-inset-bottom)`
- 最大宽度：600px，居中
- 最小高度：100vh

### 5.2 页面标题 `.page-title`

- 字号 22px、字重 **500**、字间距 1px
- 下边距 16px

### 5.3 卡片 `.jp-card`

- 背景：`var(--color-bg-card)`
- 圆角：`var(--radius-md)` = 12px
- 内边距：20px
- 阴影：`var(--shadow-md)`
- 无描边

### 5.4 卡片标题 `.jp-card-title`

- 字号 17px、字重 **500**
- 颜色：`var(--color-text)`
- 下边距 15px
- flex 布局，支持左侧 emoji 图标 `<i>`

### 5.5 信息行 `.jp-info-row` + `.jp-label` + `.jp-value`

- 标签宽度 80px，颜色 `var(--color-text-secondary)`
- 值颜色 `var(--color-text)`
- 行间距 12px

### 5.6 列表行 `.jp-list-row` + `.jp-list-left`

- 行间分隔线：`1px solid var(--color-border)`
- 行高 padding 8px
- 左侧子元素 gap 8px

### 5.7 添加行 `.jp-add-row`

- 顶部分隔线 + 上内边距 10px
- 子元素 gap 10px

### 5.8 主按钮 `.jp-btn`

- 背景：`var(--color-primary)` 藏青
- 文字：`var(--color-text-inverse)` 白
- 圆角：`var(--radius-sm)` = 6px
- 内边距：8px 18px
- 按压态：`opacity: 0.8`

### 5.9 强调按钮 `.jp-btn-accent`

- 背景：`var(--color-accent)` 金色
- 全宽、内边距 13px
- 用于 CTA / 保存 / 确认操作
- 禁用态：`opacity: 0.6`

### 5.10 胶囊切换标签 `.jp-pill` + `.jp-pill-group`

- 外形：两端圆润的胶囊形（圆角 14px），因形似药丸也称 Pill
- 默认态：透明底 + `var(--color-border)` 描边 + `var(--color-text-secondary)` 文字
- 选中态 `.active`：`var(--color-pill-active)` 底色 + 白色文字（对比度 ≥4.5:1 WCAG AA）
- 容器 `.jp-pill-group`：flex 布局、gap 8px

### 5.11 状态标签 `.status-tag`

- 进行中 `.status-pending`：`var(--color-status-pending)` 文字 + `var(--color-status-pending-bg)` 底
- 已完成 `.status-done`：`var(--color-status-completed)` 文字 + `var(--color-status-completed-bg)` 底
- 字号 11px、圆角 4px、字重 500

### 5.12 删除/危险操作 `.jp-del`

- 无背景按钮，文字色 `var(--color-danger)`
- 按压态：`var(--color-danger-bg)` 底色

### 5.13 输入框 `.jp-input-override`

- 填充底色：`var(--color-bg-input)` 暖灰
- 无边框、无聚焦描边、无聚焦阴影
- 光标色：`var(--color-accent)` 金色
- 占位符色：`var(--color-text-secondary)`
- 文字色：`var(--color-text)`

### 5.14 颜色选择器 `.color-input`

- 40×34px，圆角 8px
- 边框：`2px solid var(--color-border)`
- 原生 `<input type="color">` 样式重置

### 5.15 提示文字 `.jp-hint` / 次要文字 `.jp-muted`

- `.jp-hint`：12px、`var(--color-accent)` 金色，用于字段下方提示
- `.jp-muted`：13px、`var(--color-text-secondary)`，用于辅助说明

### 5.16 圆点 `.jp-dot`

- 12×12px 圆形，用于标签/分类前的彩色标识点

### 5.17 底部导航

- 毛玻璃背景：`var(--color-bg-nav)` + `backdrop-filter: saturate(180%) blur(20px)`
- 分割线：`0.5px solid var(--color-border)`
- ＋ 按钮：`linear-gradient(135deg, var(--color-primary), var(--color-nav-gradient-end))`
- 46×46px 圆形，阴影 `var(--color-primary-light)`
- 选中态文字色：`var(--color-primary)`
- 未选中态文字色：`var(--color-text-muted)`

### 5.18 详情页头部 `.detail-header`

- flex 布局，两端对齐
- 下边距 16px、最小高度 40px

### 5.19 优先级样式

- `.priority-high`：`var(--color-danger)` 文字 + `var(--color-danger-bg)` 底
- `.priority-medium`：`var(--color-warning)` 文字 + `var(--color-warning-bg)` 底
- `.priority-low`：`var(--color-text-secondary)` 文字

### 5.20 过渡动画

| 名称 | 效果 | 时长 |
|------|------|------|
| `.fade-*` | 淡入淡出 | `var(--transition-normal)` = 0.25s |
| `.slide-up-*` | 上滑淡入 + translateY(20px) | `var(--transition-normal)` |

## 6. iOS / Capacitor 适配

- 安全区域通过 `env(safe-area-inset-*)` 动态适配
- `.capacitor-ios`：禁用长按菜单（`-webkit-touch-callout: none`），输入框保留文字选择
- Naive UI 消息/通知容器：顶部偏移 `env(safe-area-inset-top) + 8px` 避免刘海/灵动岛遮挡
- 最小触控区域：44×44px（`button, [role='button'], .touchable`）

## 10. 无障碍 / 焦点态

所有可聚焦元素均添加 `:focus-visible` 样式，确保键盘导航可视：

```css
:focus-visible {
  outline: 2px solid var(--color-primary);
  outline-offset: 2px;
}
```

对比度标准（WCAG AA）：

| 场景 | 要求 | 当前 |
|------|------|------|
| 主文字 on card | ≥4.5:1 | ✅ `#5d574f` on `#fffdf9` ≈ 5.2:1 |
| 次要文字 on card | ≥3:1（大字）| ✅ `#857f75` on `#fffdf9` ≈ 3.5:1 |
| Pill 白字 on active | ≥4.5:1 | ✅ `#fff` on `#7a6e5e` ≈ 4.9:1 |
| Warning on card | ≥3:1 | ✅ `#b8923e` on `#fffdf9` ≈ 3.8:1 |
| Danger on card | ≥3:1 | ✅ `#bf6f65` on `#fffdf9` ≈ 3.6:1 |
| Success on card | ≥3:1 | ✅ `#7a9e6e` on `#fffdf9` ≈ 3.0:1 |

## 11. 开发约定

1. **所有颜色必须走 CSS 变量**，禁止在页面 `<style>` 中直接写 `#hex` / `rgb()` / `hsl()`
2. **新增颜色** → 先在 `_classic.scss` 中定义变量，然后在页面中通过 `var(--color-xxx)` 引用
3. **Naive UI 组件色** → 同步更新 `nuxt.config.ts` 中的 `themeConfig.light` / `themeConfig.dark`
4. **透明度变体** → 使用 `rgba(var(--color-primary-rgb), 0.15)` 模式，而非直接写带透明度的 hex
5. **深色模式** → 每个新变量都需要在 `.dark` 块中定义对应值，不可遗漏
6. **特殊模块豁免**：
   - `pages/admin/*` — 后台独立配色体系，不受经典皮肤约束
   - `pages/period/*` — 经期追踪使用粉色 `#ec4899` 为领域特征色
