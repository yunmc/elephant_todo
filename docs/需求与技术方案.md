## 一、项目概述

Elephant Todo 是一款面向个人日常管理的 **移动优先（Mobile-First）** Web 应用。主要使用场景为手机端，同时兼容平板和 PC 浏览器。支持 PWA（渐进式 Web 应用），可添加到手机主屏幕，获得接近原生 App 的体验。

采用 **SSR（服务端渲染）** 架构，基于 **Nuxt 3** 框架。相比纯客户端渲染（SPA），SSR 的优势：

- **更快的首屏加载** — 服务端直接输出 HTML，移动端弱网环境下体验显著提升
- **更好的 SEO** — 搜索引擎可直接抓取完整页面内容
- **更好的移动端性能** — 减少客户端 JavaScript 解析压力，低端手机也能流畅使用

## 二、技术栈

| 层级     | 技术                                     |
| -------- | ---------------------------------------- |
| 前端框架 | Nuxt 3（Vue 3 + TypeScript）— SSR 模式   |
| UI 组件库 | Naive UI（通过 nuxt-naiveui 模块集成）  |
| 状态管理 | Pinia（Nuxt 内置集成）                   |
| 后端 API | Nuxt 3 Nitro Server (server/ 目录)        |
| 数据库   | MySQL 8 (mysql2/promise)                 |
| 认证方式 | JWT（access + refresh token, cookie 存储）|
| 智能建议 | DeepSeek / Gemini LLM（通过提示词模板匹配） |
| 邮件服务 | Nodemailer（密码重置邮件发送）           |
| PWA      | @vite-pwa/nuxt 模块                     |
| 单元测试 | Vitest                                   |
| E2E 测试 | Playwright                               |
| 渲染模式 | SSR（服务端渲染）+ 客户端水合（Hydration）|

## 五、架构说明

### 5.1 SSR 架构概览

```
用户（手机浏览器 / PWA）
    │
    ▼
Nuxt 3 全栈服务器（Node.js + Nitro）
    │  ① 服务端渲染 HTML + 客户端水合（Hydration）
    │  ② 页面路由、布局、中间件
    │  ③ server/ 目录下的 RESTful API 接口
    │  ④ JWT 认证、业务逻辑
    │
    ▼
MySQL 数据库
```

- **Nuxt 3 全栈架构**：前端 SSR 渲染与后端 API 在同一 Nitro 服务器中运行
- **server/ 目录**：基于文件的 API 路由，Nitro 自动处理请求分发
- 无需独立 Express 服务器，部署为单一 Node.js 进程

## 七、非功能需求

| 项目       | 说明                                         |
| ---------- | -------------------------------------------- |
| 安全性     | 密码 bcrypt 加密存储；JWT 认证；SQL 注入防护；密码本端到端加密（AES-256-GCM + PBKDF2） |
| 性能       | 列表分页加载；接口响应时间 < 500ms；SSR 首屏 TTFB < 800ms；FCP < 1.5s（移动网络） |
| 移动体验   | 触控友好；动画流畅（60fps）；适配安全区域（刘海屏/底部横条） |
| PWA        | Service Worker 离线缓存；Web App Manifest；支持安装到主屏幕 |
| 可维护性   | TypeScript 全栈类型安全；代码规范（ESLint）  |
| 测试       | 单元测试（Vitest）；E2E 端到端测试（Playwright） |
| 部署       | Nuxt 3 全栈 SSR 服务（Nitro + Node.js）；可用 PM2 管理进程 |
