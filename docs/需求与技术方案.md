# Elephant Todo App — 需求与技术方案文档

> 版本：v1.0  
> 日期：2026-02-25  

---

## 一、项目概述

Elephant Todo 是一款面向个人日常任务管理的 **移动优先（Mobile-First）** Web 应用。主要使用场景为手机端，同时兼容平板和 PC 浏览器。用户可以随时随地管理自己的待办事项，支持多设备数据同步、分类标签管理、优先级设置等功能。支持 PWA（渐进式 Web 应用），可添加到手机主屏幕，获得接近原生 App 的体验。

采用 **SSR（服务端渲染）** 架构，基于 **Nuxt 3** 框架。相比纯客户端渲染（SPA），SSR 的优势：

- **更快的首屏加载** — 服务端直接输出 HTML，移动端弱网环境下体验显著提升
- **更好的 SEO** — 搜索引擎可直接抓取完整页面内容
- **更好的移动端性能** — 减少客户端 JavaScript 解析压力，低端手机也能流畅使用

## 二、技术栈

| 层级     | 技术                                     |
| -------- | ---------------------------------------- |
| 前端框架 | Nuxt 3（Vue 3 + TypeScript）— SSR 模式   |
| UI 组件库 | Naive UI（通过 nuxt-naiveui 模块集成）  |
| 状态管理 | Pinia（Nuxt 内置集成）                   |

| 单元测试 | Vitest                                   |
| E2E 测试 | Playwright                               |
| 渲染模式 | SSR（服务端渲染）+ 客户端水合（Hydration）|

### 3.2 统一输入与智能分流

App 提供一个**统一输入入口**（底部导航“添加”按钮），用户随时输入内容（文字或语音），然后由系统智能匹配 + 用户选择决定内容去向。

**交互流程：**

```
用户输入（文字 / 语音）
       │
       ▼
系统通过 LLM（DeepSeek/Gemini）智能匹配已有 Todo
       │
       ├── 找到相似 Todo → 提示用户选择：
       │       ├─ “关联到已有 Todo” → 作为随手记关联到该 Todo
       │       ├─ “新建 Todo”        → 创建为新的 Todo
       │       └─ “保存为随手记”    → 保存为独立的随手记
       │
       └── 未找到相似 → 提示用户选择：
               ├─ “新建 Todo”        → 创建为新的 Todo
               └─ “保存为随手记”    → 保存为独立的随手记
```

| 功能         | 说明                                       |
| ------------ | ------------------------------------------ |
| 统一输入     | 底部导航“添加”按钮唤起输入框，支持文字和语音 |
| 语音输入     | 长按麦克风按钮，语音识别自动转为文字。采用策略模式封装，默认使用 Web Speech API，后续可切换 |
| 智能匹配     | 输入后调用 DeepSeek/Gemini LLM，通过提示词模板与已有 Todo 进行语义匹配，返回相似 Todo 列表及分流建议 |
| 用户分流     | 用户根据匹配结果选择：关联已有 Todo / 新建 Todo / 保存为随手记 |

### 3.4 随手记（Idea）

| 功能         | 说明                                       |
| ------------ | ------------------------------------------ |
| 记录随手记   | 通过统一输入入口创建，或在随手记列表页直接新建 |

| 独立存在     | 随手记可以暂不关联 Todo，后续再关联或转化 |
| 转化为 Todo  | 未关联的随手记可以一键转化为新 Todo     |
| 关联 Todo 显示 | 随手记列表中，已关联的随手记显示关联的 Todo 标题，点击可跳转到 Todo 详情；未关联的显示“未关联”标签 |
| 随手记列表   | 独立的随手记列表页，查看所有随手记，按时间排序 |

### 3.7 密码本模块

密码本用于安全管理用户的各类账号密码。采用**端到端加密（E2E）**，密码在客户端加密后传输和存储，服务器无法解密，确保最高安全性。

| 功能         | 说明                                       |
| ------------ | ------------------------------------------ |
| 新增密码     | 保存网站/应用名称、网址、用户名、密码、备注 |
| 编辑/删除   | 修改或删除已保存的密码记录         |
| 分组管理     | 按类型分组（如：社交、银行、工作、购物等） |
| 搜索         | 按名称、网址、用户名搜索                 |
| 密码生成器   | 自动生成随机强密码，可配置长度、字符类型（大小写/数字/特殊字符） |
| 端到端加密 | 敏感字段（用户名、密码、备注）打包为 JSON 后使用 AES-256-GCM 整体加密为单一密文，密钥由用户登录密码通过 PBKDF2 派生，服务器仅存储密文 |
| 查看密码     | 默认隐藏，点击显示，支持点击复制到剪贴板（30秒后自动清除） |

> ⚠️ **密码重置警告**：密码本加密密钥由用户登录密码派生。若用户通过“忘记密码”重置登录密码，密码本数据将无法解密（旧密钥丢失）。这是端到端加密的固有限制，需在 UI 上明确提示用户。通过“修改密码”（需输入旧密码）方式则不受影响，客户端可自动用旧密钥解密后重新加密。

## 六、架构说明

### 6.1 SSR 架构概览

```
用户（手机浏览器 / PWA）
    │
    ▼
Nuxt 3 全栈服务器（Node.js + Nitro）
    │  ① 服务端渲染 HTML + 客户端水合（Hydration）
    │  ② 页面路由、布局、中间件
    │  ③ server/ 目录下的 RESTful API 接口
    │  ④ JWT 认证、业务逻辑
    │
    ▼
MySQL 数据库
```

- **Nuxt 3 全栈架构**：前端 SSR 渲染与后端 API 在同一 Nitro 服务器中运行
- **server/ 目录**：基于文件的 API 路由，Nitro 自动处理请求分发
- 无需独立 Express 服务器，部署为单一 Node.js 进程

### 6.2 项目目录结构


## 七、页面规划

| 页面         | 路由路径       | 说明                     |
| ------------ | -------------- | ------------------------ |
| 登录页       | /login         | 邮箱 + 密码登录         |
| 注册页       | /register      | 用户名 + 邮箱 + 密码注册|
| 忘记密码     | /forgot-password | 输入邮箱，发送重置链接 |
| 重置密码     | /reset-password  | 通过邮件链接设置新密码 |
| 主页（Todo 列表） | /         | Todo 列表，含搜索/筛选/分页 |
| Todo 详情    | /todo/:id      | 查看/编辑单个 Todo + 子任务管理 |
| 随手记列表   | /ideas         | 随手记列表，按时间排序     |
| 随手记详情   | /ideas/:id     | 查看/编辑单个随手记       |
| 密码本       | /vault         | 密码列表，含分组/搜索     |
| 密码详情     | /vault/:id     | 查看/编辑单个密码记录   |
| 记账         | /finance       | 记账记录列表 + 月度统计 + 分类管理 |
| 重要日期     | /important-dates | 重要日期列表 + 倒计时显示 |
| 经期追踪     | /period        | 经期记录 + 周期预测 + 易孕期预测 |
| 更多         | /more          | 功能入口 Hub（密码本/记账/重要日期/经期/设置） |
| 设置页       | /settings      | 主题切换 + 分类管理 + 标签管理 + 修改密码 |

## 八、非功能需求

| 项目       | 说明                                         |
| ---------- | -------------------------------------------- |
| 安全性     | 密码 bcrypt 加密存储；JWT 认证；SQL 注入防护；密码本端到端加密（AES-256-GCM + PBKDF2） |
| 性能       | 列表分页加载；接口响应时间 < 500ms；SSR 首屏 TTFB < 800ms；FCP < 1.5s（移动网络） |
| 移动体验   | 触控友好；动画流畅（60fps）；适配安全区域（刘海屏/底部横条） |
| PWA        | Service Worker 离线缓存；Web App Manifest；支持安装到主屏幕 |
| 可维护性   | TypeScript 全栈类型安全；代码规范（ESLint）  |
| 测试       | 单元测试（Vitest）；E2E 端到端测试（Playwright） |
| 部署       | Nuxt 3 全栈 SSR 服务（Nitro + Node.js）；可用 PM2 管理进程 |

## 九、自动化测试方案

### 9.1 后端 API 测试

| 项目     | 说明                                 |
| -------- | ------------------------------------ |
| 框架     | Vitest                               |
| 测试范围 | 类型检查、API 契约验证、加密逻辑     |
| 覆盖范围   | 所有 API 接口的正常流程 + 异常分支       |

**测试用例覆盖：**

| 模块       | 测试内容                                     |
| ---------- | ------------------------------------------ |
| 认证       | 注册成功/失败；登录成功/失败；Token 刷新；未认证拦截；忘记密码发送邮件；重置密码（Token 有效/过期/已使用）；修改密码（旧密码正确/错误） |
| Todo       | CRUD；完成/恢复；分页；筛选；权限验证（不能操作别人的 Todo） |
| 随手记     | CRUD；关联/解除关联 Todo；转化为 Todo       |
| 智能匹配   | 相似 Todo 检索；无相似结果时返回空；Embedding 服务不可用时降级处理 |
| 密码本     | 分组 CRUD；密码记录 CRUD；加密字段不可明文返回   |
| 记账       | 分类 CRUD；记录 CRUD；月度统计；按分类汇总     |
| 重要日期   | CRUD；倒计时计算；每年重复日期处理             |
| 经期追踪   | CRUD；周期自动计算；预测算法；易孕期计算       |
| 分类/标签 | CRUD；唯一性约束；级联删除验证             |

### 9.2 E2E 端到端测试

| 项目     | 说明                                 |
| -------- | ------------------------------------ |
| 框架     | Playwright                           |
| 浏览器   | Chromium（Mobile Chrome + Desktop Chrome） |
| 设备模拟 | Pixel 5（移动端）+ Desktop Chrome（桌面端）|

**测试场景：**

| 场景               | 测试内容                                   |
| ------------------ | ---------------------------------------- |
| 用户注册登录       | 注册 → 登录 → 验证首页跳转；错误密码拦截   |
| 忘记/重置密码     | 忘记密码 → 输入邮箱 → 验证发送成功提示；重置密码页设置新密码 → 登录验证 |
| 统一输入智能分流   | 输入内容 → 选择新建 Todo；输入内容 → 选择保存为随手记；输入相似内容 → 关联已有 Todo |
| Todo 完整流程     | 新建 → 编辑 → 完成 → 筛选查看 → 删除    |
| 随手记与 Todo 关联 | 创建随手记 → 关联到 Todo → Todo 详情页查看关联随手记 → 点击跳转验证 |
| 密码本操作       | 新建分组 → 新建密码 → 查看/复制 → 删除   |
| 底部导航         | 各 Tab 切换，验证页面跳转正确           |
| 暗色模式         | 切换主题，验证页面渲染正常               |
