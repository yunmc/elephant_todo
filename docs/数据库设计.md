# Elephant Todo App — 数据库设计文档

> 版本：v1.0  
> 日期：2026-02-25  
> 数据库：MySQL

---

## 一、ER 关系概览

```
users 1──N categories
users 1──N todos
users 1──N tags
users 1──N ideas
users 1──N vault_groups
users 1──N vault_entries
users 1──N password_reset_tokens
categories 1──N todos
todos 1──N ideas           (一个 Todo 可关联多个 Idea)
todos 1──N subtasks        (一个 Todo 可包含多个子任务)
todos N──N tags            (通过 todo_tags 关联)
vault_groups 1──N vault_entries  (一个分组下多个密码记录)
users 1──N finance_categories  (一个用户多个记账分类)
users 1──N finance_records     (一个用户多笔记账记录)
finance_categories 1──N finance_records  (一个分类下多笔记录)
users 1──N important_dates    (一个用户多个重要日期)
users 1──N period_records      (一个用户多条经期记录)
```

> **随手记 → Todo 递进关系**：随手记是轻量级的想法记录，可以独立存在，也可以关联到某个 Todo。多个随手记可以关联到同一个 Todo，体现从灵感到行动的递进过程。

## 二、表结构设计

### 2.1 用户表（users）

| 字段       | 类型         | 约束                    | 说明               |
| ---------- | ------------ | ----------------------- | ------------------ |
| id         | INT          | PRIMARY KEY, AUTO_INCREMENT | 主键，自增     |
| username   | VARCHAR(50)  | NOT NULL, UNIQUE        | 用户名，唯一       |
| email      | VARCHAR(100) | NOT NULL, UNIQUE        | 邮箱，唯一         |
| password   | VARCHAR(255) | NOT NULL                | 密码（bcrypt 加密）|
| created_at | DATETIME     | NOT NULL, DEFAULT NOW() | 创建时间           |
| updated_at | DATETIME     | NOT NULL, DEFAULT NOW() ON UPDATE NOW() | 更新时间 |

**索引**：
- `uk_users_username` — UNIQUE(username)
- `uk_users_email` — UNIQUE(email)

---

### 2.2 分类表（categories）

| 字段       | 类型         | 约束                    | 说明               |
| ---------- | ------------ | ----------------------- | ------------------ |
| id         | INT          | PRIMARY KEY, AUTO_INCREMENT | 主键，自增     |
| user_id    | INT          | NOT NULL, FOREIGN KEY   | 外键 → users.id    |
| name       | VARCHAR(50)  | NOT NULL                | 分类名称           |
| color      | VARCHAR(20)  | DEFAULT '#999999'       | 分类颜色（HEX）    |
| created_at | DATETIME     | NOT NULL, DEFAULT NOW() | 创建时间           |

**索引**：
- `idx_categories_user_id` — INDEX(user_id)
- `uk_categories_user_name` — UNIQUE(user_id, name)  *同一用户下分类名不重复*

**外键**：
- `fk_categories_user` — user_id → users(id) ON DELETE CASCADE

---

### 2.3 待办事项表（todos）

| 字段         | 类型                              | 约束                    | 说明               |
| ------------ | --------------------------------- | ----------------------- | ------------------ |
| id           | INT                               | PRIMARY KEY, AUTO_INCREMENT | 主键，自增     |
| user_id      | INT                               | NOT NULL, FOREIGN KEY   | 外键 → users.id    |
| category_id  | INT                               | FOREIGN KEY, NULL       | 外键 → categories.id（可为空）|
| title        | VARCHAR(200)                      | NOT NULL                | 标题               |
| description  | TEXT                              | NULL                    | 描述（可为空）     |
| priority     | ENUM('high','medium','low')       | NOT NULL, DEFAULT 'medium' | 优先级          |
| status       | ENUM('pending','completed')       | NOT NULL, DEFAULT 'pending' | 状态           |
| due_date     | DATETIME                          | NULL                    | 截止日期（可为空） |
| completed_at | DATETIME                          | NULL                    | 完成时间（可为空） |
| created_at   | DATETIME                          | NOT NULL, DEFAULT NOW() | 创建时间           |
| updated_at   | DATETIME                          | NOT NULL, DEFAULT NOW() ON UPDATE NOW() | 更新时间 |
| embedding    | JSON                              | NULL                    | 语义向量（Embedding API 生成，用于相似度匹配） |

**索引**：
- `idx_todos_user_status` — INDEX(user_id, status)  *按状态筛选*
- `idx_todos_user_priority` — INDEX(user_id, priority)  *按优先级筛选*
- `idx_todos_user_due_date` — INDEX(user_id, due_date)  *按截止日期筛选/排序*
- `idx_todos_title` — FULLTEXT(title, description)  *全文搜索*

**外键**：
- `fk_todos_user` — user_id → users(id) ON DELETE CASCADE
- `fk_todos_category` — category_id → categories(id) ON DELETE SET NULL

---

### 2.4 标签表（tags）

| 字段       | 类型         | 约束                    | 说明               |
| ---------- | ------------ | ----------------------- | ------------------ |
| id         | INT          | PRIMARY KEY, AUTO_INCREMENT | 主键，自增     |
| user_id    | INT          | NOT NULL, FOREIGN KEY   | 外键 → users.id    |
| name       | VARCHAR(50)  | NOT NULL                | 标签名称           |
| created_at | DATETIME     | NOT NULL, DEFAULT NOW() | 创建时间           |

**索引**：
- `idx_tags_user_id` — INDEX(user_id)
- `uk_tags_user_name` — UNIQUE(user_id, name)  *同一用户下标签名不重复*

**外键**：
- `fk_tags_user` — user_id → users(id) ON DELETE CASCADE

---

### 2.5 随手记表（ideas）

| 字段       | 类型         | 约束                    | 说明               |
| ---------- | ------------ | ----------------------- | ------------------ |
| id         | INT          | PRIMARY KEY, AUTO_INCREMENT | 主键，自增     |
| user_id    | INT          | NOT NULL, FOREIGN KEY   | 外键 → users.id    |
| todo_id    | INT          | FOREIGN KEY, NULL       | 外键 → todos.id（可为空，未关联时为 NULL） |
| content    | TEXT         | NOT NULL                | 随手记内容         |
| source     | ENUM('text','voice') | NOT NULL, DEFAULT 'text' | 来源方式（手动输入/语音输入） |
| embedding  | JSON         | NULL                    | 语义向量（Embedding API 生成） |
| created_at | DATETIME     | NOT NULL, DEFAULT NOW() | 创建时间           |
| updated_at | DATETIME     | NOT NULL, DEFAULT NOW() ON UPDATE NOW() | 更新时间 |

**索引**：
- `idx_ideas_user_id` — INDEX(user_id)
- `idx_ideas_todo_id` — INDEX(todo_id)　*查询某 Todo 关联的所有 Idea*
- `idx_ideas_user_created` — INDEX(user_id, created_at)　*按时间排序*
- `idx_ideas_content` — FULLTEXT(content)　*全文搜索*

**外键**：
- `fk_ideas_user` — user_id → users(id) ON DELETE CASCADE
- `fk_ideas_todo` — todo_id → todos(id) ON DELETE SET NULL　*删除 Todo 时，Idea 保留但解除关联*

---

### 2.6 子任务表（subtasks）

| 字段       | 类型                          | 约束                    | 说明               |
| ---------- | ----------------------------- | ----------------------- | ------------------ |
| id         | INT                           | PRIMARY KEY, AUTO_INCREMENT | 主键，自增     |
| todo_id    | INT                           | NOT NULL, FOREIGN KEY   | 外键 → todos.id    |
| title      | VARCHAR(200)                  | NOT NULL                | 子任务标题           |
| status     | ENUM('pending','completed')   | NOT NULL, DEFAULT 'pending' | 状态           |
| sort_order | INT                           | NOT NULL, DEFAULT 0     | 排序序号           |
| created_at | DATETIME                      | NOT NULL, DEFAULT NOW() | 创建时间           |
| updated_at | DATETIME                      | NOT NULL, DEFAULT NOW() ON UPDATE NOW() | 更新时间 |

**索引**：
- `idx_subtasks_todo_id` — INDEX(todo_id)  *查询某 Todo 的所有子任务*

**外键**：
- `fk_subtasks_todo` — todo_id → todos(id) ON DELETE CASCADE  *删除 Todo 时，自动删除其所有子任务*

---

### 2.7 Todo-标签关联表（todo_tags）

| 字段    | 类型 | 约束                         | 说明               |
| ------- | ---- | ---------------------------- | ------------------ |
| todo_id | INT  | NOT NULL, FOREIGN KEY        | 外键 → todos.id    |
| tag_id  | INT  | NOT NULL, FOREIGN KEY        | 外键 → tags.id     |

**主键**：
- PRIMARY KEY (todo_id, tag_id)  *联合主键*

**索引**：
- `idx_todo_tags_tag_id` — INDEX(tag_id)  *反向查询：某标签下的所有 Todo*

**外键**：
- `fk_todo_tags_todo` — todo_id → todos(id) ON DELETE CASCADE
- `fk_todo_tags_tag` — tag_id → tags(id) ON DELETE CASCADE

---

### 2.8 密码分组表（vault_groups）

| ---------- | ------------ | ----------------------- | ------------------ |

**索引**：

**外键**：
- `fk_vault_groups_user` — user_id → users(id) ON DELETE CASCADE

### 2.9 密码记录表（vault_entries）

> **重要**：该表的 `encrypted_data` 字段为 **客户端加密后的密文**（将用户名、密码、备注打包加密），服务器无法解密。

| 字段               | 类型          | 约束                    | 说明               |
| ------------------ | ------------- | ----------------------- | ------------------ |
| id                 | INT           | PRIMARY KEY, AUTO_INCREMENT | 主键，自增     |
| user_id            | INT           | NOT NULL, FOREIGN KEY   | 外键 → users.id    |
| group_id           | INT           | FOREIGN KEY, NULL       | 外键 → vault_groups.id（可为空） |
| name               | VARCHAR(200)  | NOT NULL                | 网站/应用名称（明文，用于搜索） |
| url                | VARCHAR(500)  | DEFAULT NULL            | 网址（明文，可为空）     |
| encrypted_data     | TEXT          | NOT NULL                | 加密后的敏感数据（用户名、密码、备注打包为 JSON 后整体加密为单一密文，IV嵌入在前12字节） |
| created_at         | DATETIME      | NOT NULL, DEFAULT NOW() | 创建时间           |
| updated_at         | DATETIME      | NOT NULL, DEFAULT NOW() ON UPDATE NOW() | 更新时间 |

**索引**：
- `idx_vault_entries_user_id` — INDEX(user_id)
- `idx_vault_entries_group_id` — INDEX(group_id)
- `idx_vault_entries_user_name` — INDEX(user_id, name)  *按名称搜索*

**外键**：
- `fk_vault_entries_user` — user_id → users(id) ON DELETE CASCADE
- `fk_vault_entries_group` — group_id → vault_groups(id) ON DELETE SET NULL  *删除分组时，记录保留但变为未分组*

## 三、建表 SQL

```sql
-- 创建数据库
CREATE DATABASE IF NOT EXISTS elephant_todo
  DEFAULT CHARACTER SET utf8mb4
  DEFAULT COLLATE utf8mb4_unicode_ci;

USE elephant_todo;

-- 用户表
CREATE TABLE users (
  id         INT          AUTO_INCREMENT PRIMARY KEY,
  username   VARCHAR(50)  NOT NULL,
  email      VARCHAR(100) NOT NULL,
  password   VARCHAR(255) NOT NULL,
  created_at DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY uk_users_username (username),
  UNIQUE KEY uk_users_email (email)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 分类表
CREATE TABLE categories (
  id         INT          AUTO_INCREMENT PRIMARY KEY,
  user_id    INT          NOT NULL,
  name       VARCHAR(50)  NOT NULL,
  color      VARCHAR(20)  DEFAULT '#999999',
  created_at DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY uk_categories_user_name (user_id, name),
  KEY idx_categories_user_id (user_id),
  CONSTRAINT fk_categories_user FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 待办事项表
CREATE TABLE todos (
  id           INT                               AUTO_INCREMENT PRIMARY KEY,
  user_id      INT                               NOT NULL,
  category_id  INT                               DEFAULT NULL,
  title        VARCHAR(200)                      NOT NULL,
  description  TEXT                              DEFAULT NULL,
  priority     ENUM('high','medium','low')       NOT NULL DEFAULT 'medium',
  status       ENUM('pending','completed')       NOT NULL DEFAULT 'pending',
  due_date     DATETIME                          DEFAULT NULL,
  completed_at DATETIME                          DEFAULT NULL,
  created_at   DATETIME                          NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at   DATETIME                          NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  embedding    JSON                              DEFAULT NULL,
  KEY idx_todos_user_id (user_id),
  KEY idx_todos_category_id (category_id),
  KEY idx_todos_user_status (user_id, status),
  KEY idx_todos_user_priority (user_id, priority),
  KEY idx_todos_user_due_date (user_id, due_date),
  FULLTEXT KEY idx_todos_title (title, description),
  CONSTRAINT fk_todos_user FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
  CONSTRAINT fk_todos_category FOREIGN KEY (category_id) REFERENCES categories(id) ON DELETE SET NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 子任务表
CREATE TABLE subtasks (
  id         INT                           AUTO_INCREMENT PRIMARY KEY,
  todo_id    INT                           NOT NULL,
  title      VARCHAR(200)                  NOT NULL,
  status     ENUM('pending','completed')   NOT NULL DEFAULT 'pending',
  sort_order INT                           NOT NULL DEFAULT 0,
  created_at DATETIME                      NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME                      NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  KEY idx_subtasks_todo_id (todo_id),
  CONSTRAINT fk_subtasks_todo FOREIGN KEY (todo_id) REFERENCES todos(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 标签表
CREATE TABLE tags (
  id         INT          AUTO_INCREMENT PRIMARY KEY,
  user_id    INT          NOT NULL,
  name       VARCHAR(50)  NOT NULL,
  created_at DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY uk_tags_user_name (user_id, name),
  KEY idx_tags_user_id (user_id),
  CONSTRAINT fk_tags_user FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 随手记表
CREATE TABLE ideas (
  id         INT                        AUTO_INCREMENT PRIMARY KEY,
  user_id    INT                        NOT NULL,
  todo_id    INT                        DEFAULT NULL,
  content    TEXT                       NOT NULL,
  source     ENUM('text','voice')       NOT NULL DEFAULT 'text',
  embedding  JSON                       DEFAULT NULL,
  created_at DATETIME                   NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME                   NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  KEY idx_ideas_user_id (user_id),
  KEY idx_ideas_todo_id (todo_id),
  KEY idx_ideas_user_created (user_id, created_at),
  FULLTEXT KEY idx_ideas_content (content),
  CONSTRAINT fk_ideas_user FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
  CONSTRAINT fk_ideas_todo FOREIGN KEY (todo_id) REFERENCES todos(id) ON DELETE SET NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- Todo-标签关联表
CREATE TABLE todo_tags (
  todo_id INT NOT NULL,
  tag_id  INT NOT NULL,
  PRIMARY KEY (todo_id, tag_id),
  KEY idx_todo_tags_tag_id (tag_id),
  CONSTRAINT fk_todo_tags_todo FOREIGN KEY (todo_id) REFERENCES todos(id) ON DELETE CASCADE,
  CONSTRAINT fk_todo_tags_tag FOREIGN KEY (tag_id) REFERENCES tags(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 密码分组表
CREATE TABLE vault_groups (
  id         INT          AUTO_INCREMENT PRIMARY KEY,
  user_id    INT          NOT NULL,
  name       VARCHAR(50)  NOT NULL,
  icon       VARCHAR(50)  DEFAULT NULL,
  sort_order INT          NOT NULL DEFAULT 0,
  created_at DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY uk_vault_groups_user_name (user_id, name),
```

## 四、设计说明

### 4.1 字符集
- 全库使用 `utf8mb4` + `utf8mb4_unicode_ci`，支持 emoji 表情等多字节字符。

### 4.2 外键级联策略
| 关系                     | 策略            | 说明                              |
| ------------------------ | --------------- | --------------------------------- |
| users → categories       | ON DELETE CASCADE | 删除用户时，自动删除其所有分类    |
| users → todos            | ON DELETE CASCADE | 删除用户时，自动删除其所有 Todo   |
| users → tags             | ON DELETE CASCADE | 删除用户时，自动删除其所有标签    |
| users → ideas            | ON DELETE CASCADE | 删除用户时，自动删除其所有随手记  |
| categories → todos       | ON DELETE SET NULL | 删除分类时，关联 Todo 的 category_id 置空 |
| todos → ideas            | ON DELETE SET NULL | 删除 Todo 时，随手记保留但解除关联（todo_id 置空） |
| todos → subtasks        | ON DELETE CASCADE | 删除 Todo 时，自动删除其所有子任务  |
| todos → todo_tags        | ON DELETE CASCADE | 删除 Todo 时，自动删除其标签关联  |
| tags → todo_tags         | ON DELETE CASCADE | 删除标签时，自动删除其 Todo 关联  |
| users → vault_groups     | ON DELETE CASCADE | 删除用户时，自动删除其所有密码分组 |
| users → vault_entries    | ON DELETE CASCADE | 删除用户时，自动删除其所有密码记录 |
| vault_groups → vault_entries | ON DELETE SET NULL | 删除分组时，密码记录保留但变为未分组 |
| users → password_reset_tokens | ON DELETE CASCADE | 删除用户时，自动删除其所有重置令牌 |
| users → finance_categories   | ON DELETE CASCADE | 删除用户时，自动删除其所有记账分类 |
| users → finance_records      | ON DELETE CASCADE | 删除用户时，自动删除其所有记账记录 |
| finance_categories → finance_records | ON DELETE SET NULL | 删除分类时，记录保留但分类置空 |
| users → important_dates      | ON DELETE CASCADE | 删除用户时，自动删除其所有重要日期 |
| users → period_records       | ON DELETE CASCADE | 删除用户时，自动删除其所有经期记录 |

### 4.3 索引策略
- 所有外键字段建立索引，优化 JOIN 查询性能
- 针对常用筛选场景建立复合索引（user_id + status / priority / due_date）
- 使用 FULLTEXT 索引支持标题和描述的全文搜索
- 同一用户下的分类名、标签名建立唯一索引，防止重复

### 4.4 时间字段
- `created_at` / `updated_at` 使用 `DATETIME` 类型，由数据库自动维护
- `completed_at` 在 Todo 标记完成时写入，恢复为未完成时置 NULL

### 4.5 密码本加密设计
- **端到端加密（E2E）**：密码数据在客户端加密后传输，服务器仅存储密文，无法解密
- **加密算法**：AES-256-GCM（对称加密）
- **密钥派生**：用户登录密码通过 PBKDF2 派生出加密密钥，密钥仅存在于客户端内存中
- **IV（初始化向量）**：每条记录使用唯一的随机 IV，与密文一起存储
- `name` 和 `url` 字段保持明文，用于搜索和列表展示；`encrypted_data` 为密文（将用户名、密码、备注打包为 JSON 后整体加密）

### 4.6 智能匹配设计
- **用途**：统一输入时的智能匹配，将用户输入与已有 Todo 进行语义相似度比较
- **方案**：通过 DeepSeek/Gemini LLM 提示词模板进行语义匹配，返回相似 Todo 列表及分流建议
- **embedding 字段**：todos 表和 ideas 表中的 `embedding` 字段为预留字段，当前智能匹配由 LLM 直接完成
- **认证**：通过 API Key 认证，Key 存储在服务端环境变量中
