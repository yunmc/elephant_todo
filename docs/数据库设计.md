```
users (根表)
 │
 ├── password_reset_tokens.user_id   → ON DELETE CASCADE
 │
 ├── categories.user_id              → ON DELETE CASCADE

 ├── todos.user_id                   → ON DELETE CASCADE
 │   ├── subtasks.todo_id            → ON DELETE CASCADE

 │   └── todo_tags.todo_id           → ON DELETE CASCADE
 │
 ├── tags.user_id                    → ON DELETE CASCADE

 │
 ├── ideas.user_id                   → ON DELETE CASCADE
 │
 ├── vault_groups.user_id            → ON DELETE CASCADE
 │   └── vault_entries.group_id      → ON DELETE SET NULL
 │
 ├── vault_entries.user_id           → ON DELETE CASCADE
 │
 ├── finance_categories.user_id      → ON DELETE CASCADE
 │   └── finance_records.category_id → ON DELETE SET NULL
 │
 ├── finance_records.user_id         → ON DELETE CASCADE
 │
 ├── important_dates.user_id         → ON DELETE CASCADE

 └── period_records.user_id          → ON DELETE CASCADE
```

## 三、外键级联策略

| 外键名 | 表 | 字段 | 引用 | 删除策略 |
|--------|-----|------|------|---------|
| fk_reset_tokens_user | password_reset_tokens | user_id | users(id) | CASCADE |
| fk_categories_user | categories | user_id | users(id) | CASCADE |
| fk_tags_user | tags | user_id | users(id) | CASCADE |
| fk_todos_user | todos | user_id | users(id) | CASCADE |
| fk_todos_category | todos | category_id | categories(id) | SET NULL |
| fk_subtasks_todo | subtasks | todo_id | todos(id) | CASCADE |
| fk_todo_tags_todo | todo_tags | todo_id | todos(id) | CASCADE |
| fk_todo_tags_tag | todo_tags | tag_id | tags(id) | CASCADE |
| fk_ideas_user | ideas | user_id | users(id) | CASCADE |
| fk_ideas_todo | ideas | todo_id | todos(id) | SET NULL |
| fk_vault_groups_user | vault_groups | user_id | users(id) | CASCADE |
| fk_vault_entries_user | vault_entries | user_id | users(id) | CASCADE |
| fk_vault_entries_group | vault_entries | group_id | vault_groups(id) | SET NULL |
| fk_finance_categories_user | finance_categories | user_id | users(id) | CASCADE |
| fk_finance_records_user | finance_records | user_id | users(id) | CASCADE |
| fk_finance_records_category | finance_records | category_id | finance_categories(id) | SET NULL |
| fk_important_dates_user | important_dates | user_id | users(id) | CASCADE |
| fk_period_records_user | period_records | user_id | users(id) | CASCADE |

### 级联策略说明

- **CASCADE**：父记录删除时，子记录一并删除。用于强依赖关系（如用户删除 → 全部数据清除）。
- **SET NULL**：父记录删除时，子记录的外键字段置为 NULL。用于弱关联关系（如分类删除 → Todo 的 category_id 置 NULL，Todo 不删除）。

## 五、通用设计说明

### 5.1 时间字段

- 所有表统一使用 `created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP`
- 需要记录修改时间的表使用 `updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP`
- 日期类型字段（`due_date`、`record_date`、`start_date`、`end_date`、`date`）使用 `DATE` 类型

### 5.2 多租户隔离

- 每张业务表均包含 `user_id` 字段
- 所有 API 查询均通过 `WHERE user_id = ?` 实现数据隔离
- 所有外键 `user_id` 均设置 `ON DELETE CASCADE`，用户注销时一并清除全部数据

### 5.3 软删除 vs 硬删除

- 当前所有表采用 **硬删除**（DELETE），无 `deleted_at` 字段
- 用户删除通过级联自动处理

### 5.4 密码本加密设计

- `vault_entries.encrypted_data` 字段存储 **AES-256-GCM** 加密的密文
- 加密密钥由用户主密码经 **PBKDF2**（10万次迭代）派生
- **IV**（12 字节）嵌入密文前部
- 服务端仅存储密文，**无法解密**（端到端加密）
- 详见 [modules/密码本.md](modules/密码本.md)

### 5.5 智能匹配设计

- `todos.embedding` 和 `ideas.embedding` 字段存储 JSON 格式的语义向量
- 当前使用 LLM 提示词模板进行语义匹配，embedding 为预留字段
- 详见 [modules/智能匹配.md](modules/智能匹配.md)
