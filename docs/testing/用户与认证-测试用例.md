# 用户与认证 — 测试用例

> 所属项目：Elephant Todo App  
> 测试框架：Vitest  
> 最后更新：2026-02-26

---

## 一、单元测试 / API 测试

### 1.1 注册

| 编号 | 用例 | 预期结果 |
| ---- | ---- | -------- |
| AUTH-R01 | 正常注册（用户名 + 邮箱 + 密码） | 201，返回用户信息 + Token |
| AUTH-R02 | 用户名已存在 | 400/409，提示用户名已被使用 |
| AUTH-R03 | 邮箱已存在 | 400/409，提示邮箱已被注册 |
| AUTH-R04 | 密码太短（< 6 位） | 400，参数校验失败 |
| AUTH-R05 | 缺少必填字段（用户名/邮箱/密码） | 400，参数校验失败 |
| AUTH-R06 | 邮箱格式不合法 | 400，参数校验失败 |

### 1.2 登录

| 编号 | 用例 | 预期结果 |
| ---- | ---- | -------- |
| AUTH-L01 | 正确邮箱 + 密码登录 | 200，返回 access_token + refresh_token（cookie） |
| AUTH-L02 | 邮箱不存在 | 401，认证失败 |
| AUTH-L03 | 密码错误 | 401，认证失败 |
| AUTH-L04 | 缺少邮箱或密码 | 400，参数校验失败 |

### 1.3 Token 刷新

| 编号 | 用例 | 预期结果 |
| ---- | ---- | -------- |
| AUTH-T01 | 携带有效 refresh_token 刷新 | 200，返回新的 access_token |
| AUTH-T02 | refresh_token 过期/无效 | 401，刷新失败 |
| AUTH-T03 | 无 refresh_token | 401 |

### 1.4 获取当前用户

| 编号 | 用例 | 预期结果 |
| ---- | ---- | -------- |
| AUTH-M01 | 携带有效 Token 访问 /api/auth/me | 200，返回用户信息 |
| AUTH-M02 | 未携带 Token | 401 |
| AUTH-M03 | Token 过期 | 401 |

### 1.5 忘记密码

| 编号 | 用例 | 预期结果 |
| ---- | ---- | -------- |
| AUTH-F01 | 输入已注册邮箱 | 200，发送重置邮件 |
| AUTH-F02 | 输入未注册邮箱 | 200（不泄露邮箱是否存在） |
| AUTH-F03 | 缺少邮箱 | 400 |

### 1.6 重置密码

| 编号 | 用例 | 预期结果 |
| ---- | ---- | -------- |
| AUTH-P01 | 有效 Token + 新密码 | 200，密码重置成功 |
| AUTH-P02 | Token 已过期 | 400，Token 无效 |
| AUTH-P03 | Token 已使用 | 400，Token 无效 |
| AUTH-P04 | 无效 Token | 400 |

### 1.7 修改密码

| 编号 | 用例 | 预期结果 |
| ---- | ---- | -------- |
| AUTH-C01 | 旧密码正确 + 新密码 | 200，密码修改成功 |
| AUTH-C02 | 旧密码错误 | 400/401 |
| AUTH-C03 | 未登录 | 401 |

## 二、认证中间件测试

| 编号 | 用例 | 预期结果 |
| ---- | ---- | -------- |
| AUTH-MW01 | 未登录访问受保护页面 | 重定向到 /login |
| AUTH-MW02 | 已登录访问 /login | 重定向到 / |
| AUTH-MW03 | Token 过期后访问受保护 API | 自动刷新 Token 后重试 |
