## 一、单元测试 / API 测试

### 1.1 经期记录 CRUD

| 编号 | 用例 | 预期结果 |
| ---- | ---- | -------- |
| PR-C01 | 创建记录（start_date） | 201 |
| PR-C02 | 创建记录（start_date + end_date + 流量 + 症状 + 心情） | 201，所有字段正确保存 |
| PR-C03 | 缺少 start_date | 400 |
| PR-C04 | end_date 早于 start_date | 400 |

| PR-U01 | 更新记录（修改 end_date） | 200 |
| PR-D01 | 删除记录 | 200 |

### 1.2 自动周期计算

| 编号 | 用例 | 预期结果 |
| ---- | ---- | -------- |
| PR-A01 | 创建第 2 条记录后 | 第 1 条记录自动计算 cycle_length |
| PR-A02 | 有 end_date 的记录 | 自动计算 period_length = end - start + 1 |
| PR-A03 | 无 end_date 的记录 | period_length 为 NULL |
| PR-A04 | 更新 start_date | 相关记录的 cycle_length 重新计算 |
| PR-A05 | 删除记录 | 相邻记录的 cycle_length 重新计算 |
| PR-A06 | 仅 1 条记录 | cycle_length 为 NULL（无法计算） |

### 1.3 预测算法

| 编号 | 用例 | 预期结果 |
| ---- | ---- | -------- |
| PP-01 | 有 6+ 条记录时预测 | 基于最近 6 条平均周期长度预测 |
| PP-02 | 有 2-5 条记录时预测 | 基于已有记录的平均值预测 |
| PP-03 | 仅 1 条记录时预测 | 使用默认周期长度 28 天预测 |
| PP-04 | 无记录时预测 | 返回 null + 提示消息 |
| PP-05 | 预测结果包含易孕窗口 | fertile_window = 排卵日（下次经期前 14 天）± 2 天 |
| PP-06 | 预测结果包含平均值 | average_cycle_length + average_period_length |

### 1.4 症状与心情

| 编号 | 用例 | 预期结果 |
| ---- | ---- | -------- |
| PR-S01 | 保存多个症状（JSON 数组） | 正确存储 ["痛经","头痛","腹胀"] |
| PR-S02 | 空症状列表 | symptoms 为 null 或 [] |
| PR-S03 | 保存心情 | mood 字段正确存储 |
