# 每日打卡 — 测试用例

> 所属项目：Elephant Todo App  
> 测试框架：Vitest  
> 最后更新：2026-03-09

---

## 一、单元测试 / API 测试

### 1.1 习惯项 CRUD

| 编号 | 用例 | 预期结果 |
| ---- | ---- | -------- |
| CL-C01 | 创建习惯项（标题 + 图标） | 201，返回新习惯项，is_active=1，sort_order=0 |
| CL-C02 | 仅标题创建习惯项 | 201，icon 默认 ✅ |
| CL-C03 | 缺少标题 | 400，参数校验失败 |
| CL-C04 | 标题超过 100 字符 | 400 |
| CL-R01 | 获取习惯项列表 | 200，按 sort_order 升序返回，含已暂停项 |
| CL-R02 | 新用户获取列表 | 200，返回空列表 |
| CL-U01 | 更新习惯项标题 | 200，标题更新成功 |
| CL-U02 | 更新习惯项图标 | 200，图标更新成功 |
| CL-U03 | 暂停习惯项（is_active=0） | 200，is_active 变为 0 |
| CL-U04 | 启用习惯项（is_active=1） | 200，is_active 变为 1 |
| CL-U05 | 更新不存在的习惯项 | 404 |
| CL-D01 | 删除习惯项 | 200，删除成功 |
| CL-D02 | 删除不存在的习惯项 | 404 |
| CL-D03 | 删除习惯项后级联删除打卡记录 | 打卡记录被自动删除 |

### 1.2 排序

| 编号 | 用例 | 预期结果 |
| ---- | ---- | -------- |
| CL-O01 | 批量更新排序（正常 itemIds） | 200，sort_order 按传入顺序更新 |
| CL-O02 | 传入不属于当前用户的 itemId | 400 或仅更新自己的 |
| CL-O03 | 空数组 | 400 |

### 1.3 今日打卡状态

| 编号 | 用例 | 预期结果 |
| ---- | ---- | -------- |
| CL-T01 | 获取今日打卡状态（未打卡） | 200，所有启用项 checked=false |
| CL-T02 | 获取今日打卡状态（部分打卡） | 200，已打卡项 checked=true，未打卡项 checked=false |
| CL-T03 | 获取今日打卡状态（全部打卡） | 200，所有项 checked=true，checked_at 非空 |
| CL-T04 | 暂停的习惯不出现在今日列表 | 200，不包含 is_active=0 的习惯项 |
| CL-T05 | 传入 date 参数查看历史某天 | 200，返回该日期的打卡状态 |

### 1.4 打卡操作

| 编号 | 用例 | 预期结果 |
| ---- | ---- | -------- |
| CK-C01 | 正常打卡（item_id + date） | 200，打卡成功，checked_at 非空 |
| CK-C02 | 不传 date，默认今天 | 200，check_date 为今天 |
| CK-C03 | 重复打卡同一天同一项 | 200（幂等），不产生重复记录 |
| CK-C04 | 打卡不存在的习惯项 | 404 |
| CK-C05 | 打卡其他用户的习惯项 | 404 |
| CK-C06 | 补打历史日期 | 200，check_date 为指定日期 |
| CK-D01 | 取消打卡 | 200，记录被删除 |
| CK-D02 | 取消未打卡的记录 | 200（幂等），无副作用 |
| CK-D03 | 取消其他用户的打卡 | 404 |

### 1.5 历史记录

| 编号 | 用例 | 预期结果 |
| ---- | ---- | -------- |
| CL-H01 | 查询日期范围内的历史记录 | 200，按日期倒序返回，每天包含各项打卡状态 |
| CL-H02 | 无打卡记录的日期范围 | 200，返回空列表或日期列表中都是 checked=false |
| CL-H03 | 不传 start_date / end_date | 400 或使用默认范围（如最近 7 天） |
| CL-H04 | start_date 晚于 end_date | 400 |

### 1.6 统计数据

| 编号 | 用例 | 预期结果 |
| ---- | ---- | -------- |
| CL-S01 | 获取统计（days=30） | 200，返回 overall + items + heatmap |
| CL-S02 | 连续打卡 3 天后查看 streak | current_streak = 3 |
| CL-S03 | 连续打卡 5 天，中断 1 天，再打卡 2 天 | current_streak = 2，longest_streak = 5 |
| CL-S04 | 今天未打卡，昨天打卡了 | current_streak = 0（从今天算起需连续） |
| CL-S05 | 某习惯 30 天内打卡 20 天 | completion_rate ≈ 66.7% |
| CL-S06 | 新创建的习惯只计算创建后的天数 | total_days 从创建日期起算，非整个统计范围 |
| CL-S07 | 暂停的习惯不计入统计 | 暂停期间不计入 total_days |
| CL-S08 | 无习惯时查看统计 | 200，返回空统计（无 items，overall 为零值） |
| CL-S09 | heatmap 数据格式正确 | 每天包含 date、total、checked、rate 字段 |
| CL-S10 | days=7 / days=90 参数 | 200，返回对应范围的统计数据 |

## 三、边界与异常测试

| 编号 | 用例 | 预期结果 |
| ---- | ---- | -------- |
| CL-E01 | 同一天对同一习惯重复 POST /check | 幂等处理，不报错也不重复插入 |
| CL-E02 | 删除唯一的习惯项后查看今日列表 | 200，返回空列表 |
| CL-E03 | 大量习惯项（50+）性能 | 响应时间合理（< 500ms） |
| CL-E04 | 长时间跨度统计（days=365） | 正确返回，无超时 |
| CL-E05 | 未来日期打卡 | 400，不允许打卡未来日期 |
| CL-E06 | 非法 date 格式 | 400 |
| CL-E07 | item_id 为非数字 | 400 |
