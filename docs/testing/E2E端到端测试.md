# E2E 端到端测试用例

> 所属项目：Elephant Todo App  
> 测试框架：Playwright  
> 浏览器：Chromium（Mobile Chrome + Desktop Chrome）  
> 设备模拟：Pixel 5（移动端）+ Desktop Chrome（桌面端）  
> 最后更新：2026-02-27

---

## 基础设施

### 运行方式

```bash
# 确保开发服务器已启动 (端口 3001)
npm run dev

# 运行 E2E 测试
npm run test:e2e

# 运行指定文件
npx playwright test tests/e2e/auth.spec.ts

# UI 模式调试
npx playwright test --ui
```

### 配置说明

| 项目 | 值 |
|------|-----|
| baseURL | `http://localhost:3001` |
| 测试目录 | `tests/e2e/` |
| 共享 fixtures | `tests/e2e/fixtures/auth.fixture.ts` |
| 测试账号 | 每次运行自动注册临时用户（无需手动配置） |
| 并行策略 | 文件间并行，文件内串行 |
| 超时 | 单测 30s，导航 15s |

### 文件结构

```
tests/e2e/
├── fixtures/
│   └── auth.fixture.ts     # 自动注册+登录 fixture，提供 authedPage
├── auth.spec.ts             # 用户认证（注册/登录/错误/登出/表单验证）
├── todo.spec.ts             # Todo CRUD + 子任务 + 筛选 + 优先级/分类/标签
├── ideas.spec.ts            # 随手记 CRUD + 关联/取消关联/转为待办 + 搜索
├── vault.spec.ts            # 密码本完整流程（解锁/分组/条目CRUD/加密解密）
├── finance.spec.ts          # 记账 CRUD + 分类管理 + 月度统计
├── important-dates.spec.ts  # 重要日期 CRUD + 倒计时 + 每年重复
├── period.spec.ts           # 经期记录 CRUD + 预测
├── navigation.spec.ts       # 底部导航 + 更多页面 + 快速添加
├── settings.spec.ts         # 分类管理 + 标签管理 + 修改密码 + 主题
└── theme.spec.ts            # 主题切换 + 持久化
```

### 选择器策略

项目未使用 `data-testid`，采用以下策略：

| 优先级 | 选择器 | 示例 |
|--------|--------|------|
| 1 | `getByRole('button', { name })` | 按钮操作 |
| 2 | `getByPlaceholder(...)` | 输入框 |
| 3 | `getByText(...)` | 标签/Tab/链接文字 |
| 4 | `locator('.css-class')` | 结构元素（`.todo-item`, `.record-card` 等） |

| E2E-A15 | 重复邮箱注册 | 注册用户 → 用相同邮箱再次注册 | 显示错误提示，停留在注册页 |
| E2E-A17 | 已登录访问登录页重定向 | 注入认证 Cookie → 访问 `/login` | 自动重定向到首页 |
