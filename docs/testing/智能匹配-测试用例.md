# 智能匹配 — 测试用例

> 所属项目：Elephant Todo App  
> 测试框架：Vitest  
> 最后更新：2026-02-26

---

## 一、单元测试 / API 测试

### 1.1 智能建议

| 编号 | 用例 | 预期结果 |
| ---- | ---- | -------- |
| MATCH-01 | 输入与已有 Todo 相似的文本 | 200，返回相似 Todo 列表 + 分流建议 |
| MATCH-02 | 输入与已有 Todo 无关的文本 | 200，返回空相似列表 + 建议新建 |
| MATCH-03 | 空文本输入 | 400，参数校验失败 |
| MATCH-04 | 未登录 | 401 |
| MATCH-05 | LLM 服务不可用时降级 | 200，返回空列表（不报错），降级为不匹配 |

### 1.2 LLM Provider 切换

| 编号 | 用例 | 预期结果 |
| ---- | ---- | -------- |
| MATCH-06 | 仅配置 Gemini API Key | 使用 Gemini 进行匹配 |
| MATCH-07 | 仅配置 DeepSeek API Key | 使用 DeepSeek 进行匹配 |
| MATCH-08 | 两个 Key 都配置 | 优先使用配置的首选 Provider |
| MATCH-09 | 两个 Key 都未配置 | 返回降级结果（空列表） |

## 二、语音输入测试

| 编号 | 用例 | 预期结果 |
| ---- | ---- | -------- |
| VOICE-01 | 浏览器支持 Web Speech API | 正常识别语音并转为文字 |
| VOICE-02 | 浏览器不支持 Web Speech API | 显示不支持提示，禁用语音按钮 |
| VOICE-03 | 语音识别超时 | 自动停止，显示超时提示 |
