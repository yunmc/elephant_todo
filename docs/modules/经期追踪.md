# 经期追踪模块

> 所属项目：Elephant Todo App  
> 最后更新：2026-02-26

---

## 一、功能说明

经期追踪模块用于记录经期数据，并基于历史数据预测下次经期和易孕窗口。

| 功能         | 说明                                       |
| ------------ | ------------------------------------------ |
| 经期记录     | 记录每次经期的开始/结束日期              |
| 流量记录     | 记录经期流量（轻/中/重）                  |
| 症状记录     | 记录伴随症状（痛经、头痛、腹胀、疲劳、腰痛、恶心、失眠、痘痘） |
| 多人记录     | 支持为不同对象记录经期（通过 person_name 区分，默认"我"） |
| 周期预测     | 基于最近 6 次周期数据计算平均周期长度，预测下次经期开始/结束日期 |
| 易孕期预测   | 基于预测的排卵日（下次经期前 14 天）±2 天计算易孕窗口 |
| 自动计算     | 自动计算每次经期的周期长度和持续天数（按 person_name 分组） |

### 预测算法

```
1. 获取用户最近 6 条有 cycle_length 的记录
2. 计算平均周期长度 avgCycleLength（无数据时默认 28 天）
3. 计算平均经期长度 avgPeriodLength（无数据时默认 5 天）
4. 下次经期开始 = 最后一次经期开始日期 + avgCycleLength
5. 下次经期结束 = 下次经期开始 + avgPeriodLength - 1
6. 排卵日 = 下次经期开始 - 14 天
7. 易孕窗口 = 排卵日 ± 2 天
```

### 自动周期计算

每次创建/更新/删除经期记录后，系统自动重新计算**同一 person_name** 下所有记录的：
- `cycle_length`：当前记录的 start_date 与下一条记录的 start_date 之间的天数
- `period_length`：end_date - start_date + 1（仅在 end_date 存在时计算）

采用 CASE 批量 UPDATE 一次性更新，避免 N+1 查询。

## 二、API 接口

| 方法   | 路径                    | 说明               | 认证 |
| ------ | ----------------------- | ------------------ | ---- |
| GET    | /api/period             | 获取经期记录列表（支持 ?person_name 筛选） | 是 |
| POST   | /api/period             | 创建经期记录       | 是   |
| PUT    | /api/period/:id         | 更新经期记录（字段白名单） | 是   |
| DELETE | /api/period/:id         | 删除经期记录       | 是   |
| GET    | /api/period/predict     | 获取经期预测（支持 ?person_name 筛选） | 是 |
| GET    | /api/period/persons     | 获取当前用户的记录对象名称列表 | 是 |

## 三、数据库表

### 3.1 经期记录表（period_records）

| 字段          | 类型                              | 约束                    | 说明               |
| ------------- | --------------------------------- | ----------------------- | ------------------ |
| id            | INT                               | PRIMARY KEY, AUTO_INCREMENT | 主键，自增     |
| user_id       | INT                               | NOT NULL, FOREIGN KEY   | 外键 → users.id    |
| person_name   | VARCHAR(50)                       | NOT NULL, DEFAULT '我'  | 记录对象名称（支持多人） |
| start_date    | DATE                              | NOT NULL                | 经期开始日期       |
| end_date      | DATE                              | DEFAULT NULL            | 经期结束日期       |
| cycle_length  | INT                               | DEFAULT NULL            | 周期长度（自动计算） |
| period_length | INT                               | DEFAULT NULL            | 经期持续天数（自动计算） |
| flow_level    | ENUM('light','moderate','heavy')  | DEFAULT 'moderate'      | 流量等级           |
| symptoms      | JSON                              | DEFAULT NULL            | 症状列表（JSON 数组） |
| note          | TEXT                              | DEFAULT NULL            | 备注               |
| created_at    | DATETIME                          | NOT NULL, DEFAULT NOW() | 创建时间           |
| updated_at    | DATETIME                          | NOT NULL, DEFAULT NOW() ON UPDATE NOW() | 更新时间 |

**索引**：
- `idx_period_records_user_id` — INDEX(user_id)
- `idx_period_records_user_date` — INDEX(user_id, start_date)
- `idx_period_records_user_person` — INDEX(user_id, person_name)

**外键**：
- `fk_period_records_user` — user_id → users(id) ON DELETE CASCADE

---

## 四、建表 SQL

```sql
CREATE TABLE period_records (
  id            INT                               AUTO_INCREMENT PRIMARY KEY,
  user_id       INT                               NOT NULL,
  person_name   VARCHAR(50)                       NOT NULL DEFAULT '我' COMMENT '记录对象名称（支持多人）',
  start_date    DATE                              NOT NULL,
  end_date      DATE                              DEFAULT NULL,
  cycle_length  INT                               DEFAULT NULL,
  period_length INT                               DEFAULT NULL,
  flow_level    ENUM('light','moderate','heavy')  DEFAULT 'moderate',
  symptoms      JSON                              DEFAULT NULL,
  note          TEXT                              DEFAULT NULL,
  created_at    DATETIME                          NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at    DATETIME                          NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  KEY idx_period_records_user_id (user_id),
  KEY idx_period_records_user_date (user_id, start_date),
  KEY idx_period_records_user_person (user_id, person_name),
  CONSTRAINT fk_period_records_user FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```
