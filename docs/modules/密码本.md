# 密码本模块

> 所属项目：Elephant Todo App  
> 最后更新：2026-02-26

---

## 一、功能说明

密码本用于安全管理用户的各类账号密码。采用**端到端加密（E2E）**，密码在客户端加密后传输和存储，服务器无法解密，确保最高安全性。

| 功能         | 说明                                       |
| ------------ | ------------------------------------------ |
| 新增密码     | 保存网站/应用名称、网址、用户名、密码、备注 |
| 编辑/删除   | 修改或删除已保存的密码记录         |
| 分组管理     | 按类型分组（如：社交、银行、工作、购物等） |
| 搜索         | 按名称、网址、用户名搜索                 |
| 密码生成器   | 自动生成随机强密码，可配置长度、字符类型（大小写/数字/特殊字符） |
| 端到端加密 | 敏感字段（用户名、密码、备注）打包为 JSON 后使用 AES-256-GCM 整体加密为单一密文，密钥由用户登录密码通过 PBKDF2 派生，服务器仅存储密文 |
| 查看密码     | 默认隐藏，点击显示，支持点击复制到剪贴板（30秒后自动清除） |

> ⚠️ **密码重置警告**：密码本加密密钥由用户登录密码派生。若用户通过"忘记密码"重置登录密码，密码本数据将无法解密（旧密钥丢失）。这是端到端加密的固有限制，需在 UI 上明确提示用户。通过"修改密码"（需输入旧密码）方式则不受影响，客户端可自动用旧密钥解密后重新加密。

### 加密设计

- **加密算法**：AES-256-GCM（对称加密）
- **密钥派生**：用户登录密码通过 PBKDF2 派生出加密密钥，密钥仅存在于客户端内存中
- **IV（初始化向量）**：每条记录使用唯一的随机 IV，与密文一起存储
- `name` 和 `url` 字段保持明文，用于搜索和列表展示；`encrypted_data` 为密文

## 三、数据库表

### 3.1 密码分组表（vault_groups）

| 字段       | 类型         | 约束                    | 说明               |
| ---------- | ------------ | ----------------------- | ------------------ |
| id         | INT          | PRIMARY KEY, AUTO_INCREMENT | 主键，自增     |
| user_id    | INT          | NOT NULL, FOREIGN KEY   | 外键 → users.id    |
| name       | VARCHAR(50)  | NOT NULL                | 分组名称           |
| icon       | VARCHAR(50)  | DEFAULT NULL            | 分组图标（可选）   |
| sort_order | INT          | NOT NULL, DEFAULT 0     | 排序序号           |
| created_at | DATETIME     | NOT NULL, DEFAULT NOW() | 创建时间           |

**索引**：
- `idx_vault_groups_user_id` — INDEX(user_id)
- `uk_vault_groups_user_name` — UNIQUE(user_id, name)

**外键**：
- `fk_vault_groups_user` — user_id → users(id) ON DELETE CASCADE

### 3.2 密码记录表（vault_entries）

> **重要**：`encrypted_data` 字段为客户端加密后的密文，服务器无法解密。

| 字段               | 类型          | 约束                    | 说明               |
| ------------------ | ------------- | ----------------------- | ------------------ |
| id                 | INT           | PRIMARY KEY, AUTO_INCREMENT | 主键，自增     |
| user_id            | INT           | NOT NULL, FOREIGN KEY   | 外键 → users.id    |
| group_id           | INT           | FOREIGN KEY, NULL       | 外键 → vault_groups.id（可为空） |
| name               | VARCHAR(200)  | NOT NULL                | 网站/应用名称（明文） |
| url                | VARCHAR(500)  | DEFAULT NULL            | 网址（明文）       |
| encrypted_data     | TEXT          | NOT NULL                | 加密敏感数据（IV 嵌入前 12 字节） |
| created_at         | DATETIME      | NOT NULL, DEFAULT NOW() | 创建时间           |
| updated_at         | DATETIME      | NOT NULL, DEFAULT NOW() ON UPDATE NOW() | 更新时间 |

**索引**：
- `idx_vault_entries_user_id` — INDEX(user_id)
- `idx_vault_entries_group_id` — INDEX(group_id)
- `idx_vault_entries_user_name` — INDEX(user_id, name)

**外键**：
- `fk_vault_entries_user` — user_id → users(id) ON DELETE CASCADE
- `fk_vault_entries_group` — group_id → vault_groups(id) ON DELETE SET NULL

## 四、建表 SQL

```sql
CREATE TABLE vault_groups (
  id         INT          AUTO_INCREMENT PRIMARY KEY,
  user_id    INT          NOT NULL,
  name       VARCHAR(50)  NOT NULL,
  icon       VARCHAR(50)  DEFAULT NULL,
  sort_order INT          NOT NULL DEFAULT 0,
  created_at DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY uk_vault_groups_user_name (user_id, name),
  KEY idx_vault_groups_user_id (user_id),
  CONSTRAINT fk_vault_groups_user FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE TABLE vault_entries (
  id                 INT           AUTO_INCREMENT PRIMARY KEY,
  user_id            INT           NOT NULL,
  group_id           INT           DEFAULT NULL,
  name               VARCHAR(200)  NOT NULL,
  url                VARCHAR(500)  DEFAULT NULL,
  encrypted_data     TEXT          NOT NULL COMMENT 'IV嵌入在加密数据中(前12字节)',
  created_at         DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at         DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  KEY idx_vault_entries_user_id (user_id),
  KEY idx_vault_entries_group_id (group_id),
  KEY idx_vault_entries_user_name (user_id, name),
  CONSTRAINT fk_vault_entries_user FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
  CONSTRAINT fk_vault_entries_group FOREIGN KEY (group_id) REFERENCES vault_groups(id) ON DELETE SET NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```
