# 记账模块

> 所属项目：Elephant Todo App  
> 最后更新：2026-02-26

---

## 一、功能说明

记账模块用于记录日常收支，帮助用户了解自己的消费情况。

| 功能         | 说明                                       |
| ------------ | ------------------------------------------ |
| 收支记录     | 记录每笔收入/支出，包含金额、分类、日期、备注 |
| 分类管理     | 自定义收支分类（如：餐饮、交通、工资等），支持图标和排序 |
| 月度统计     | 按月查看总收入、总支出、结余              |
| 分类统计     | 按分类汇总支出/收入金额                  |
| 筛选查看     | 按月份、收支类型、分类筛选记录            |
| 分页加载     | 记录列表支持分页                          |

## 三、数据库表

### 3.1 记账分类表（finance_categories）

| 字段       | 类型                            | 约束                    | 说明               |
| ---------- | ------------------------------- | ----------------------- | ------------------ |
| id         | INT                             | PRIMARY KEY, AUTO_INCREMENT | 主键，自增     |
| user_id    | INT                             | NOT NULL, FOREIGN KEY   | 外键 → users.id    |
| name       | VARCHAR(50)                     | NOT NULL                | 分类名称           |
| icon       | VARCHAR(50)                     | DEFAULT '💰'           | 分类图标           |
| type       | ENUM('income','expense')        | NOT NULL, DEFAULT 'expense' | 收入/支出类型     |
| sort_order | INT                             | NOT NULL, DEFAULT 0     | 排序序号           |
| created_at | DATETIME                        | NOT NULL, DEFAULT NOW() | 创建时间           |

**索引**：
- `idx_finance_categories_user_id` — INDEX(user_id)
- `uk_finance_categories_user_name_type` — UNIQUE(user_id, name, type)

**外键**：
- `fk_finance_categories_user` — user_id → users(id) ON DELETE CASCADE

### 3.2 记账记录表（finance_records）

| 字段        | 类型                            | 约束                    | 说明               |
| ----------- | ------------------------------- | ----------------------- | ------------------ |
| id          | INT                             | PRIMARY KEY, AUTO_INCREMENT | 主键，自增     |
| user_id     | INT                             | NOT NULL, FOREIGN KEY   | 外键 → users.id    |
| category_id | INT                             | FOREIGN KEY, NULL       | 外键 → finance_categories.id |
| type        | ENUM('income','expense')        | NOT NULL                | 收入/支出类型     |
| amount      | DECIMAL(12,2)                   | NOT NULL                | 金额               |
| note        | VARCHAR(500)                    | DEFAULT NULL            | 备注               |
| record_date | DATE                            | NOT NULL                | 记账日期           |
| created_at  | DATETIME                        | NOT NULL, DEFAULT NOW() | 创建时间           |
| updated_at  | DATETIME                        | NOT NULL, DEFAULT NOW() ON UPDATE NOW() | 更新时间 |

**索引**：
- `idx_finance_records_user_id` — INDEX(user_id)
- `idx_finance_records_category_id` — INDEX(category_id)
- `idx_finance_records_user_date` — INDEX(user_id, record_date)
- `idx_finance_records_user_type` — INDEX(user_id, type)

**外键**：
- `fk_finance_records_user` — user_id → users(id) ON DELETE CASCADE
- `fk_finance_records_category` — category_id → finance_categories(id) ON DELETE SET NULL

## 四、建表 SQL

```sql
CREATE TABLE finance_categories (
  id         INT                          AUTO_INCREMENT PRIMARY KEY,
  user_id    INT                          NOT NULL,
  name       VARCHAR(50)                  NOT NULL,
  icon       VARCHAR(50)                  DEFAULT '💰',
  type       ENUM('income','expense')     NOT NULL DEFAULT 'expense',
  sort_order INT                          NOT NULL DEFAULT 0,
  created_at DATETIME                     NOT NULL DEFAULT CURRENT_TIMESTAMP,
  KEY idx_finance_categories_user_id (user_id),
  UNIQUE KEY uk_finance_categories_user_name_type (user_id, name, type),
  CONSTRAINT fk_finance_categories_user FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE TABLE finance_records (
  id          INT                          AUTO_INCREMENT PRIMARY KEY,
  user_id     INT                          NOT NULL,
  category_id INT                          DEFAULT NULL,
  type        ENUM('income','expense')     NOT NULL,
  amount      DECIMAL(12,2)                NOT NULL,
  note        VARCHAR(500)                 DEFAULT NULL,
  record_date DATE                         NOT NULL,
  created_at  DATETIME                     NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at  DATETIME                     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  KEY idx_finance_records_user_id (user_id),
  KEY idx_finance_records_category_id (category_id),
  KEY idx_finance_records_user_date (user_id, record_date),
  KEY idx_finance_records_user_type (user_id, type),
  CONSTRAINT fk_finance_records_user FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
  CONSTRAINT fk_finance_records_category FOREIGN KEY (category_id) REFERENCES finance_categories(id) ON DELETE SET NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```
