# 每日打卡模块

> 所属项目：Elephant Todo App  
> 最后更新：2026-03-09

---

## 一、功能说明

每日打卡模块用于管理固定的日常习惯，每天自动展示待打卡列表，支持一键打卡/取消，并提供历史记录和连续打卡统计。

### 1.1 习惯管理

| 功能         | 说明                                       |
| ------------ | ------------------------------------------ |
| 创建习惯     | 标题（必填）、图标（emoji，可选，默认 ✅）  |
| 编辑习惯     | 修改习惯的标题和图标                       |
| 删除习惯     | 删除习惯及其所有打卡记录（CASCADE）        |
| 排序         | 支持手动调整习惯的显示顺序                 |
| 暂停/启用    | 暂停的习惯不出现在每日打卡列表中，不计入统计，但保留历史数据 |

### 1.2 每日打卡

| 功能         | 说明                                       |
| ------------ | ------------------------------------------ |
| 今日打卡列表 | 展示所有启用的习惯项，显示是否已打卡       |
| 一键打卡     | 点击即完成打卡，支持乐观更新（UI 立即响应） |
| 取消打卡     | 误操作可取消当天的打卡                     |
| 查看其他日期 | 可切换日期查看/补打历史打卡                |
| 完成进度     | 顶部显示当日完成进度（如 3/5）             |

### 1.3 统计与历史

| 功能           | 说明                                       |
| -------------- | ------------------------------------------ |
| 日历热力图     | 类似 GitHub 贡献图，颜色深浅表示每日完成度 |
| 当前连续天数   | 从今天往回数，连续打卡的天数               |
| 最长连续天数   | 历史最长连续打卡记录                       |
| 各项完成率     | 指定时间范围内，每个习惯的打卡天数/总天数  |
| 时间范围筛选   | 支持查看最近 7 天 / 30 天 / 90 天的统计    |

### 统计算法

```
连续天数计算：
1. 获取指定习惯（或全部习惯）的打卡记录，按 check_date DESC 排序
2. 从今天开始往回遍历，如果当天有打卡则 streak++，否则终止
3. 若指定了具体习惯，只看该习惯；若查总体，则要求当天所有启用习惯都已打卡才算连续

完成率计算：
1. 确定时间范围（start_date ~ end_date）
2. 计算范围内的有效天数（习惯创建日期之后的天数）
3. 完成率 = 打卡天数 / 有效天数 × 100%
```

## 三、数据库表

### 3.1 习惯项表（checklist_items）

| 字段       | 类型         | 约束                          | 说明                 |
| ---------- | ------------ | ----------------------------- | -------------------- |
| id         | INT          | PRIMARY KEY, AUTO_INCREMENT   | 主键，自增           |
| user_id    | INT          | NOT NULL, FOREIGN KEY         | 外键 → users.id      |
| title      | VARCHAR(100) | NOT NULL                      | 习惯名称             |
| icon       | VARCHAR(20)  | DEFAULT '✅'                  | 图标（emoji）        |
| sort_order | INT          | NOT NULL, DEFAULT 0           | 排序序号             |
| is_active  | TINYINT(1)   | NOT NULL, DEFAULT 1           | 是否启用（1=启用 0=暂停） |
| created_at | DATETIME     | NOT NULL, DEFAULT NOW()       | 创建时间             |
| updated_at | DATETIME     | NOT NULL, DEFAULT NOW() ON UPDATE NOW() | 更新时间   |

**索引**：
- `idx_checklist_items_user` — INDEX(user_id, is_active, sort_order)

**外键**：
- `fk_checklist_items_user` — user_id → users(id) ON DELETE CASCADE

### 3.2 打卡记录表（checklist_records）

| 字段       | 类型     | 约束                          | 说明                         |
| ---------- | -------- | ----------------------------- | ---------------------------- |
| id         | INT      | PRIMARY KEY, AUTO_INCREMENT   | 主键，自增                   |
| item_id    | INT      | NOT NULL, FOREIGN KEY         | 外键 → checklist_items.id    |
| user_id    | INT      | NOT NULL, FOREIGN KEY         | 外键 → users.id（冗余，便于查询） |
| check_date | DATE     | NOT NULL                      | 打卡日期                     |
| checked_at | DATETIME | NOT NULL, DEFAULT NOW()       | 实际打卡时间                 |

**约束**：
- `uq_checklist_records_item_date` — UNIQUE(item_id, check_date) 防止同天重复打卡

**索引**：
- `idx_checklist_records_user_date` — INDEX(user_id, check_date)
- `idx_checklist_records_item_date` — INDEX(item_id, check_date)

**外键**：
- `fk_checklist_records_item` — item_id → checklist_items(id) ON DELETE CASCADE
- `fk_checklist_records_user` — user_id → users(id) ON DELETE CASCADE

## 四、建表 SQL

```sql
CREATE TABLE checklist_items (
  id         INT          AUTO_INCREMENT PRIMARY KEY,
  user_id    INT          NOT NULL,
  title      VARCHAR(100) NOT NULL,
  icon       VARCHAR(20)  DEFAULT '✅',
  sort_order INT          NOT NULL DEFAULT 0,
  is_active  TINYINT(1)   NOT NULL DEFAULT 1 COMMENT '1=启用 0=暂停',
  created_at DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  KEY idx_checklist_items_user (user_id, is_active, sort_order),
  CONSTRAINT fk_checklist_items_user FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE TABLE checklist_records (
  id         INT      AUTO_INCREMENT PRIMARY KEY,
  item_id    INT      NOT NULL,
  user_id    INT      NOT NULL,
  check_date DATE     NOT NULL,
  checked_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY uq_checklist_records_item_date (item_id, check_date),
  KEY idx_checklist_records_user_date (user_id, check_date),
  KEY idx_checklist_records_item_date (item_id, check_date),
  CONSTRAINT fk_checklist_records_item FOREIGN KEY (item_id) REFERENCES checklist_items(id) ON DELETE CASCADE,
  CONSTRAINT fk_checklist_records_user FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

## 五、页面结构

### 5.1 今日打卡页（主页面）

- **路由**：`/checklist`
- **入口**：「更多」页面卡片，图标 ☑️
- **布局**：
  - 顶部：当前日期 + 完成进度条（如 3/5）
  - 中间：打卡列表 — 每行显示 emoji + 标题 + 打卡复选框
  - 底部：「管理习惯」按钮 + 「查看统计」按钮
- **交互**：
  - 点击复选框即打卡（乐观更新，UI 立即响应）
  - 再次点击可取消打卡
  - 支持左右滑动或日期选择器切换日期
  - 查看过去日期时可补打卡

### 5.2 习惯管理页

- **路由**：`/checklist/manage`
- **功能**：
  - 列表展示所有习惯项（含已暂停的，灰色标识）
  - 新增习惯：输入标题 + 选择 emoji
  - 编辑习惯：点击进入编辑
  - 删除习惯：左滑或确认弹窗
  - 排序：上下箭头调整顺序
  - 暂停/启用：开关切换

### 5.3 统计页

- **路由**：`/checklist/stats`
- **功能**：
  - 日历热力图：按月展示，颜色深浅表示当日完成度
  - 总体统计卡片：完美天数（全部完成的天数）、总体完成率、当前连续天数
  - 各项统计列表：每个习惯的完成率、当前连续 / 最长连续
  - 时间范围切换：最近 7 天 / 30 天 / 90 天

## 六、设计决策

| 决策 | 理由 |
| ---- | ---- |
| 独立模块，非 Todo 子功能 | 习惯打卡「每日重置 + 连续追踪」与 Todo「一次性任务」本质不同 |
| check_date 使用 DATE 类型 | 方便按天聚合查询和唯一约束 |
| user_id 冗余存在 records 表 | 统计查询可直接按 user_id + check_date 查，避免每次 JOIN checklist_items |
| is_active 软标记暂停 | 暂停不删除，保留历史打卡数据完整性 |
| streak 实时计算不存储 | 避免数据不一致，打卡/取消打卡时无需维护额外字段 |
| UNIQUE(item_id, check_date) | 数据库层面防止同天重复打卡，比应用层检查更可靠 |
| 一期只支持「每天」频率 | 简化实现，后续可在 checklist_items 加 frequency 字段扩展 |
