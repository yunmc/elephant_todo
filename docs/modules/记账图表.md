# 记账图表模块

> 所属项目：Elephant Todo App  
> 最后更新：2026-03-12

---

## 一、功能概览

为记账模块新增可视化图表页面，帮助用户直观了解财务状况。图表页面作为记账的子页面，通过 Tab 切换访问。

### 图表列表

| 序号 | 图表名称 | 类型 | 说明 | 优先级 |
| ---- | -------- | ---- | ---- | ------ |
| 1 | 支出分类饼图 | 饼图 / 环形图 | 展示当月各分类支出占比 | P0 |
| 2 | 月度收支趋势 | 折线图 | 展示近 6/12 个月的收入、支出、结余趋势 | P0 |
| 3 | 收入分类饼图 | 饼图 / 环形图 | 展示当月各分类收入占比 | P1 |
| 4 | 预算执行对比 | 进度条 / 横向柱状图 | 各分类预算 vs 实际支出对比（依赖预算功能上线） | P2 |

## 二、图表详细设计

### 2.1 支出分类饼图

**用途**：展示选定月份中，各支出分类的金额与占比。

**数据来源**：复用现有 `GET /api/finance/statistics` 返回的 `by_category` 字段，筛选 `type === 'expense'` 的数据。

**展示规则**：
- 使用环形图（Doughnut），中间显示当月总支出金额
- 每个扇区显示分类图标 + 名称 + 百分比
- 金额为 0 的分类不显示
- 分类超过 8 个时，将排名靠后的合并为"其他"
- 点击扇区可高亮对应分类，下方显示该分类详情
- 无数据时显示空状态："本月暂无支出记录"

**配色方案**：
- 使用预设的 8 色调色板，按金额降序分配
- 调色板：`['#5B8FF9', '#5AD8A6', '#F6BD16', '#E86452', '#6DC8EC', '#945FB9', '#FF9845', '#1E9493']`

**交互**：
- 支持月份切换（复用已有月份选择器）
- 点击图例/扇区切换选中状态

---

### 2.2 月度收支趋势折线图

**用途**：展示近几个月的收入、支出走势，帮助发现消费规律。

**数据来源**：需要**新增 API** `GET /api/finance/statistics/trend`。

**展示规则**：
- 默认显示近 6 个月，可切换为近 12 个月
- 两条折线：收入（绿色 `#5AD8A6`）、支出（红色 `#E86452`）
- X 轴：月份（如 "1月", "2月"）
- Y 轴：金额，自动适配范围
- 数据点显示圆点，点击/hover 展示具体数值 tooltip
- 可选：增加结余面积填充区（收入 - 支出的差值区域）

**交互**：
- 切换 6/12 个月范围
- 点击数据点跳转到该月记账详情（可选）

---

### 2.3 收入分类饼图

**用途**：展示选定月份中，各收入分类的金额与占比。

**数据来源**：复用 `GET /api/finance/statistics` 的 `by_category`，筛选 `type === 'income'`。

**展示规则**：与支出分类饼图一致，中心显示总收入金额，配色使用绿色系调色板。

---

### 2.4 预算执行对比（P2）

**用途**：对比各分类预算目标与实际支出。

**数据来源**：复用 `GET /api/finance/budgets/progress`。

**展示规则**：
- 横向柱状图或进度条列表
- 每行：分类图标 + 名称 + 进度条（实际/预算）+ 百分比
- 超过 80% 黄色警告，超过 100% 红色超支

## 三、新增 API 接口

### 3.1 月度趋势统计

| 方法 | 路径 | 说明 | 认证 |
| ---- | ---- | ---- | ---- |
| GET | /api/finance/statistics/trend | 获取月度收支趋势 | 是 |

**请求参数**（Query）：

| 参数 | 类型 | 必填 | 说明 |
| ---- | ---- | ---- | ---- |
| months | number | 否 | 查询月数，默认 6，最大 12 |

**响应示例**：

```json
{
  "success": true,
  "data": [
    { "month": "2025-10", "income": 8000, "expense": 5200, "balance": 2800 },
    { "month": "2025-11", "income": 8000, "expense": 4800, "balance": 3200 },
    { "month": "2025-12", "income": 9500, "expense": 6100, "balance": 3400 },
    { "month": "2026-01", "income": 8000, "expense": 5500, "balance": 2500 },
    { "month": "2026-02", "income": 8200, "expense": 4900, "balance": 3300 },
    { "month": "2026-03", "income": 8000, "expense": 3200, "balance": 4800 }
  ]
}
```

**SQL 实现思路**：

```sql
SELECT
  DATE_FORMAT(record_date, '%Y-%m') AS month,
  COALESCE(SUM(CASE WHEN type = 'income' THEN amount ELSE 0 END), 0) AS income,
  COALESCE(SUM(CASE WHEN type = 'expense' THEN amount ELSE 0 END), 0) AS expense
FROM finance_records
WHERE user_id = ?
  AND record_date >= ? -- 起始月1号
  AND record_date <= ? -- 当月最后一天
GROUP BY DATE_FORMAT(record_date, '%Y-%m')
ORDER BY month ASC
```

> 注意：查询范围内没有记录的月份需要前端补零（或后端填充空月份）。

## 四、技术选型

### 图表库：ECharts

选择 ECharts 的理由：
- 功能丰富，饼图/折线图/柱状图均原生支持
- 移动端交互体验好（触摸缩放、tooltip 跟随）
- 体积可按需引入（tree-shaking）
- 社区活跃，文档完善

**安装**：

```bash
npm install echarts
```

**Nuxt 集成方式**：
- 仅在客户端渲染图表（`<ClientOnly>` 包裹或动态 import）
- 封装通用组件 `components/charts/BaseChart.vue` 处理 resize / 主题 / 销毁

## 六、Store 扩展

在 `stores/finance.ts` 中新增：

```typescript
// ==================== Chart Statistics ====================
const trend = ref<{ month: string; income: number; expense: number; balance: number }[]>([])

async function fetchTrend(months: number = 6) {
  const res = await api.get<typeof trend.value>('/finance/statistics/trend', { months })
  trend.value = res.data || []
}
```

## 七、皮肤 / 深色模式适配

- 图表背景色使用 `transparent`，跟随卡片背景
- 文字颜色读取 CSS 变量 `--color-text-primary`、`--color-text-secondary`
- 图表线条 / 网格颜色读取 `--color-border`
- 深色模式下自动切换浅色文字 + 暗色网格
