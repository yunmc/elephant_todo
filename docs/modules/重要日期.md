# 重要日期模块

> 所属项目：Elephant Todo App  
> 最后更新：2026-02-26

---

## 一、功能说明

重要日期模块用于记录和管理用户的重要纪念日、生日等日期，并提供倒计时功能。

| 功能         | 说明                                       |
| ------------ | ------------------------------------------ |
| 日期记录     | 记录重要日期（标题、日期、图标、备注）    |
| 倒计时显示   | 自动计算距每个日期的天数，按最近优先排序  |
| 农历支持     | 可标记为农历日期                          |
| 每年重复     | 支持设置为每年重复的纪念日                |
| 提醒设置     | 可设置提前提醒天数                        |

---

## 二、API 接口

| 方法   | 路径                         | 说明               | 认证 |
| ------ | ---------------------------- | ------------------ | ---- |
| GET    | /api/important-dates         | 获取重要日期列表（含倒计时，按最近排序） | 是 |
| POST   | /api/important-dates         | 创建重要日期       | 是   |
| PUT    | /api/important-dates/:id     | 更新重要日期       | 是   |
| DELETE | /api/important-dates/:id     | 删除重要日期       | 是   |

### API 特殊逻辑

- **GET 列表**：服务端自动计算每条记录距今天的天数（`days_until`），对于 `repeat_yearly` 的记录会找到下一个发生日期
- **排序**：按 `days_until` 升序排列，最近的日期排在前面

## 三、数据库表

### 3.1 重要日期表（important_dates）

| 字段               | 类型         | 约束                    | 说明               |
| ------------------ | ------------ | ----------------------- | ------------------ |
| id                 | INT          | PRIMARY KEY, AUTO_INCREMENT | 主键，自增     |
| user_id            | INT          | NOT NULL, FOREIGN KEY   | 外键 → users.id    |
| title              | VARCHAR(200) | NOT NULL                | 日期标题           |
| date               | DATE         | NOT NULL                | 日期               |
| is_lunar           | BOOLEAN      | NOT NULL, DEFAULT FALSE | 是否为农历日期     |
| repeat_yearly      | BOOLEAN      | NOT NULL, DEFAULT TRUE  | 是否每年重复       |
| remind_days_before | INT          | NOT NULL, DEFAULT 0     | 提前提醒天数       |
| icon               | VARCHAR(50)  | DEFAULT '📅'           | 图标               |
| note               | TEXT         | DEFAULT NULL            | 备注               |
| created_at         | DATETIME     | NOT NULL, DEFAULT NOW() | 创建时间           |
| updated_at         | DATETIME     | NOT NULL, DEFAULT NOW() ON UPDATE NOW() | 更新时间 |

**索引**：
- `idx_important_dates_user_id` — INDEX(user_id)
- `idx_important_dates_user_date` — INDEX(user_id, date)

**外键**：
- `fk_important_dates_user` — user_id → users(id) ON DELETE CASCADE

## 四、建表 SQL

```sql
CREATE TABLE important_dates (
  id                 INT           AUTO_INCREMENT PRIMARY KEY,
  user_id            INT           NOT NULL,
  title              VARCHAR(200)  NOT NULL,
  date               DATE          NOT NULL,
  is_lunar           BOOLEAN       NOT NULL DEFAULT FALSE,
  repeat_yearly      BOOLEAN       NOT NULL DEFAULT TRUE,
  remind_days_before INT           NOT NULL DEFAULT 0,
  icon               VARCHAR(50)   DEFAULT '📅',
  note               TEXT          DEFAULT NULL,
  created_at         DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at         DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  KEY idx_important_dates_user_id (user_id),
  KEY idx_important_dates_user_date (user_id, date),
  CONSTRAINT fk_important_dates_user FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```
