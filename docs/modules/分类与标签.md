# 分类与标签模块

> 所属项目：Elephant Todo App  
> 最后更新：2026-02-26

---

## 一、功能说明

| 功能           | 说明                                       |
| -------------- | ------------------------------------------ |
| 创建分类       | 用户自定义分类（如：工作、学习、生活）     |
| 编辑/删除分类  | 管理已有分类                              |
| 为 Todo 指定分类 | 每个 Todo 可归属于一个分类             |
| 标签管理       | 每个 Todo 可添加多个标签                   |

## 三、数据库表

### 3.1 分类表（categories）

| 字段       | 类型         | 约束                    | 说明               |
| ---------- | ------------ | ----------------------- | ------------------ |
| id         | INT          | PRIMARY KEY, AUTO_INCREMENT | 主键，自增     |
| user_id    | INT          | NOT NULL, FOREIGN KEY   | 外键 → users.id    |
| name       | VARCHAR(50)  | NOT NULL                | 分类名称           |
| color      | VARCHAR(20)  | DEFAULT '#999999'       | 分类颜色（HEX）    |
| created_at | DATETIME     | NOT NULL, DEFAULT NOW() | 创建时间           |

**索引**：
- `idx_categories_user_id` — INDEX(user_id)
- `uk_categories_user_name` — UNIQUE(user_id, name)

**外键**：
- `fk_categories_user` — user_id → users(id) ON DELETE CASCADE

### 3.2 标签表（tags）

| 字段       | 类型         | 约束                    | 说明               |
| ---------- | ------------ | ----------------------- | ------------------ |
| id         | INT          | PRIMARY KEY, AUTO_INCREMENT | 主键，自增     |
| user_id    | INT          | NOT NULL, FOREIGN KEY   | 外键 → users.id    |
| name       | VARCHAR(50)  | NOT NULL                | 标签名称           |
| created_at | DATETIME     | NOT NULL, DEFAULT NOW() | 创建时间           |

**索引**：
- `idx_tags_user_id` — INDEX(user_id)
- `uk_tags_user_name` — UNIQUE(user_id, name)

**外键**：
- `fk_tags_user` — user_id → users(id) ON DELETE CASCADE

### 3.3 Todo-标签关联表（todo_tags）

| 字段    | 类型 | 约束                         | 说明               |
| ------- | ---- | ---------------------------- | ------------------ |
| todo_id | INT  | NOT NULL, FOREIGN KEY        | 外键 → todos.id    |
| tag_id  | INT  | NOT NULL, FOREIGN KEY        | 外键 → tags.id     |

**主键**：PRIMARY KEY (todo_id, tag_id)

**索引**：
- `idx_todo_tags_tag_id` — INDEX(tag_id)

**外键**：
- `fk_todo_tags_todo` — todo_id → todos(id) ON DELETE CASCADE
- `fk_todo_tags_tag` — tag_id → tags(id) ON DELETE CASCADE

## 四、建表 SQL

```sql
CREATE TABLE categories (
  id         INT          AUTO_INCREMENT PRIMARY KEY,
  user_id    INT          NOT NULL,
  name       VARCHAR(50)  NOT NULL,
  color      VARCHAR(20)  DEFAULT '#999999',
  created_at DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY uk_categories_user_name (user_id, name),
  KEY idx_categories_user_id (user_id),
  CONSTRAINT fk_categories_user FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE TABLE tags (
  id         INT          AUTO_INCREMENT PRIMARY KEY,
  user_id    INT          NOT NULL,
  name       VARCHAR(50)  NOT NULL,
  created_at DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY uk_tags_user_name (user_id, name),
  KEY idx_tags_user_id (user_id),
  CONSTRAINT fk_tags_user FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE TABLE todo_tags (
  todo_id INT NOT NULL,
  tag_id  INT NOT NULL,
  PRIMARY KEY (todo_id, tag_id),
  KEY idx_todo_tags_tag_id (tag_id),
  CONSTRAINT fk_todo_tags_todo FOREIGN KEY (todo_id) REFERENCES todos(id) ON DELETE CASCADE,
  CONSTRAINT fk_todo_tags_tag FOREIGN KEY (tag_id) REFERENCES tags(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```
