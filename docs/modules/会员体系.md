## 一、功能概述

会员体系是所有付费功能的基础设施。本阶段不涉及支付对接，仅建立会员状态判断、权限校验、升级引导的完整链路。

| 项目 | 说明 |
|------|------|
| 目标 | 让系统能够区分免费/付费用户，并在付费功能入口做拦截引导 |
| 不包含 | 支付对接（微信/支付宝/IAP 实际调用）、象币系统、商店 |
| 包含 | 数据库预留 iOS 自动续费字段 + Webhook 接口骨架（无实际验证逻辑） |
| 测试支持 | 通过 API 手动设置用户 plan（开发/管理员用），方便后续功能调试 |

## 四、后端实现

### 4.1 Premium 校验工具 — `server/utils/premium.ts`

```typescript
import type { H3Event } from 'h3'

export interface PremiumStatus {
  isPremium: boolean
  plan: 'free' | 'premium'
  expiresAt: string | null
  autoRenew: boolean   // 是否自动续费中
  expired: boolean     // true = 曾经是会员但已过期（且未开启自动续费）
}

/**
 * 获取用户的 Premium 状态（不报错）
 */
export async function getPremiumStatus(userId: number): Promise<PremiumStatus> {
  const [rows] = await getDb().query<any[]>(
    'SELECT plan, plan_expires_at, auto_renew FROM users WHERE id = ?',
    [userId]
  )
  const user = rows[0]
  if (!user) throw createError({ statusCode: 404, message: '用户不存在' })

  const now = new Date()
  const notExpired = user.plan_expires_at && new Date(user.plan_expires_at) > now
  // auto_renew=1 时即使过期也给宽限期（Apple billing retry 最多 60 天）
  const isPremium = user.plan === 'premium'
    && user.plan_expires_at
    && (notExpired || !!user.auto_renew)

  return {
    isPremium,
    plan: user.plan,
    expiresAt: user.plan_expires_at?.toISOString() ?? null,
    autoRenew: !!user.auto_renew,
    expired: user.plan === 'premium'
      && !!user.plan_expires_at
      && new Date(user.plan_expires_at) <= now
      && !user.auto_renew,  // 自动续费中不算「已过期」
  }
}

/**
 * 要求 Premium 权限，否则抛 403。
 * 用于付费功能 API 的前置校验。
 */
export async function requirePremium(userId: number): Promise<void> {
  const status = await getPremiumStatus(userId)
  if (!status.isPremium) {
    throw createError({
      statusCode: 403,
      message: 'PREMIUM_REQUIRED',
      data: { expired: status.expired },
    })
  }
}

/**
 * 要求 Admin 权限，否则抛 403。
 * 注意：这里查的是 admin_users 表，与 App 用户无关。
 * adminId 来自后台独立的登录体系（P7 实现）。
 */
export async function requireAdmin(adminId: number): Promise<void> {
  const [rows] = await getDb().query<any[]>(
    'SELECT id FROM admin_users WHERE id = ?',
    [adminId]
  )
  if (!rows[0]) {
    throw createError({ statusCode: 403, message: '无权限' })
  }
}
```

### 4.4 iOS 自动续费 — Webhook 接口骨架

> 本阶段只建骨架，不实现 Apple 签名验证和 JWS 解析。后续对接 IAP 时填充。

Apple 自动续费的完整生命周期由 **App Store Server Notifications V2** 驱动：

```
用户首次订阅 → Apple 扣款 → 通知我们 SUBSCRIBED
     ↓
到期前 Apple 自动续费 → 通知我们 DID_RENEW → 我们延长 plan_expires_at
     ↓
用户取消自动续费 → 通知我们 DID_CHANGE_RENEWAL_STATUS → auto_renew=0
     ↓
到期后不续费 → 通知我们 EXPIRED → plan='free'
     ↓
用户重新订阅 → 通知我们 SUBSCRIBED → 恢复 premium
```

**关键通知类型**：

| notificationType | subtype | 我们的处理 |
|------------------|---------|------------|
| SUBSCRIBED | INITIAL_BUY | 创建订单，设 plan=premium，auto_renew=1 |
| DID_RENEW | — | 创建续费订单，延长 plan_expires_at |
| DID_CHANGE_RENEWAL_STATUS | AUTO_RENEW_DISABLED | 设 auto_renew=0（会员仍有效直到到期） |
| DID_CHANGE_RENEWAL_STATUS | AUTO_RENEW_ENABLED | 设 auto_renew=1 |
| EXPIRED | VOLUNTARY | plan='free'，auto_renew=0 |
| EXPIRED | BILLING_RETRY_PERIOD | plan 暂不降级，等 Apple 重试扣款 |
| DID_FAIL_TO_RENEW | — | 记录扣款失败，保持 premium 直到 grace period 结束 |
| REFUND | — | status='refunded'，plan='free' |
| REVOKE | — | Apple 撤销（家庭共享移除等），plan='free' |

### 5.1 `composables/usePremium.ts`

```typescript
export function usePremium() {
  const authStore = useAuthStore()

  const isPremium = computed(() => {
    const user = authStore.user
    if (!user) return false
    if (user.plan !== 'premium') return false
    if (!user.plan_expires_at) return false
    const notExpired = new Date(user.plan_expires_at) > new Date()
    // auto_renew=1 时即使过期也给宽限期（Apple billing retry）
    return notExpired || user.auto_renew
  })

  const isExpired = computed(() => {
    const user = authStore.user
    if (!user) return false
    return user.plan === 'premium'
      && !!user.plan_expires_at
      && new Date(user.plan_expires_at) <= new Date()
      && !user.auto_renew  // 自动续费中不算「已过期」
  })

  const isAutoRenew = computed(() => {
    return authStore.user?.auto_renew ?? false
  })

  const daysRemaining = computed(() => {
    const user = authStore.user
    if (!isPremium.value || !user?.plan_expires_at) return 0
    const diff = new Date(user.plan_expires_at).getTime() - Date.now()
    return Math.max(0, Math.ceil(diff / (1000 * 60 * 60 * 24)))
  })

  // 显示升级弹窗的全局状态
  const showUpgradeModal = useState('show-upgrade-modal', () => false)
  const upgradeModalExpired = useState('upgrade-modal-expired', () => false)

  /** 检查是否 Premium，非会员弹出升级引导，返回 false */
  function guardPremium(): boolean {
    if (isPremium.value) return true
    upgradeModalExpired.value = isExpired.value
    showUpgradeModal.value = true
    return false
  }

  return {
    isPremium,
    isExpired,
    isAutoRenew,
    daysRemaining,
    showUpgradeModal,
    upgradeModalExpired,
    guardPremium,
  }
}
```

> **轮询逻辑**：不放在 composable 里（多组件调用会冲突），改为在 `plugins/premium-poll.client.ts` 中统一启动：

```typescript
// plugins/premium-poll.client.ts
export default defineNuxtPlugin(() => {
  const authStore = useAuthStore()
  const POLL_INTERVAL = 5 * 60 * 1000 // 5 分钟
  let timer: ReturnType<typeof setInterval> | null = null

  const start = () => {
    if (timer) return
    timer = setInterval(() => {
      if (authStore.user) authStore.fetchMe().catch(() => {})
    }, POLL_INTERVAL)
  }

  const stop = () => {
    if (timer) { clearInterval(timer); timer = null }
  }

  document.addEventListener('visibilitychange', () => {
    if (document.visibilityState === 'visible') {
      if (authStore.user) authStore.fetchMe().catch(() => {})
      start()
    } else {
      stop()
    }
  })

  // 登录后开始轮询
  watch(() => authStore.user, (user) => {
    user ? start() : stop()
  }, { immediate: true })
})
```

### 5.4 useApi 错误处理增强

在 `composables/useApi.ts` 中增加对 403 PREMIUM_REQUIRED 的全局处理：

```typescript
// 在 catch 块中，401 处理之后加：
if (error?.response?.status === 403) {
  const data = error?.response?._data || error?.data
  if (data?.message === 'PREMIUM_REQUIRED') {
    const { showUpgradeModal, upgradeModalExpired } = usePremium()
    upgradeModalExpired.value = data?.data?.expired || false
    showUpgradeModal.value = true
    throw error  // 仍然抛出，让调用方知道操作失败
  }
}
```

## 八、边界情况 & 决策

| # | 情况 | 处理 |
|---|------|------|
| 1 | 用户有多笔订单，到期时间不同 | `plan_expires_at` 以最晚的到期时间为准（续费时取 max） |
| 2 | 系统时钟偏差 | 服务端统一用 MySQL NOW()，不依赖应用服务器时间 |
| 3 | 过期瞬间用户正在操作 | 前端 `isPremium` 是实时计算的 computed，过期后下次操作自然拦截 |
| 4 | JWT payload 里要不要存 plan？ | **不存**。plan 可能随时变化（续费/过期），放 JWT 里会导致缓存不一致。每次需要判断时查 DB 或走 API |
| 5 | Premium 状态要不要缓存？ | 前端：存在 `authStore.user` 里（登录/刷新时更新）。后端 `requirePremium` 每次查 DB（一条简单 SELECT，性能不是问题） |
| 6 | 设置页 Premium 卡片的"立即升级"按钮 | 当前阶段弹 Toast 提示"支付功能开发中"，不跳转 |
| 7 | iOS 自动续费到期但 Apple 还在重试扣款（billing retry） | `auto_renew=1` 但已过期 → 前端显示“续费处理中”，不立刻降级，给 Apple 最多 60 天重试期 |
| 8 | 用户关闭自动续费后 plan_expires_at 前的行为 | 会员功能照常可用，只是卡片文案变成“将于 XX 到期（已取消）” |
| 9 | 一个 Apple 账号对应多个 App 账号 | 通过 `apple_original_transaction_id` 绑定，同一订阅链只能绑定一个 user_id |
| 10 | 「管理订阅」按钮行为 | iOS：跳转系统订阅管理页面；Web/Android：显示“请在 App Store 中管理” |

## 九、iOS 自动续费架构总览

```
┌──────────────┐    ┌──────────────┐    ┌──────────────────┐
│  iOS App     │    │  App Store   │    │  Our Server      │
│  (Capacitor) │    │  (Apple)     │    │  (Nuxt API)      │
└──────┬───────┘    └──────┬───────┘    └────────┬─────────┘
       │ 1. 发起订阅        │                     │
       │──────────────────→│                     │
       │                   │ 2. 扣款成功          │
       │ 3. 返回 Transaction│                     │
       │←──────────────────│                     │
       │ 4. 上报 txnId     │                     │
       │──────────────────────────────────────→  │
       │                   │                     │ 5. 验证+激活 Premium
       │                   │                     │
       │                   │ 6. 到期自动续费       │
       │                   │────────────────────→│
       │                   │  (Server Notification V2: DID_RENEW)
       │                   │                     │ 7. 延长 plan_expires_at
       │                   │                     │    创建续费订单
       │                   │                     │
       │                   │ 8. 用户取消自动续费   │
       │                   │────────────────────→│
       │                   │  (DID_CHANGE_RENEWAL_STATUS)
       │                   │                     │ 9. auto_renew=0
       │                   │                     │    会员仍有效至到期
```

**关键点**：
- 自动续费是 **Apple 管理的**，我们不控制续费时机
- 我们只负责 **接收通知 → 更新数据**
- 前端通过 `auto_renew` 字段展示正确的续费状态
- 用户管理（取消/恢复）订阅全部在 iOS 系统设置中操作

**App Store Connect 配置**（后续实施）：
- Product ID: `com.elephant.premium.monthly` / `com.elephant.premium.yearly`
- Subscription Group: `com.elephant.premium`
- Server Notification URL: `https://your-domain.com/api/premium/apple-webhook`
