# 底部导航「+」按钮上下文感知机制

> 所属项目：Elephant Todo App  
> 最后更新：2026-03-10

---

## 一、设计思路

底部导航栏中间的「+」按钮是全局入口，根据当前所在页面自动适配不同的新建操作，减少用户操作路径。

---

## 二、页面行为映射

| 当前路由 | 页面 | 「+」按钮行为 | 实现方式 |
|---------|------|-------------|---------|
| `/` | 待办 | 打开快速添加弹窗（新建待办 / 随手记） | Layout 默认行为 |
| `/ideas` | 随手记 | 打开快速添加弹窗（新建待办 / 随手记） | Layout 默认行为 |
| `/finance` | 记账 | 打开「记一笔」弹窗 | 页面注册 `showAddModal = true` |
| `/checklist` | 每日打卡 | 跳转到习惯管理页 | 页面注册 `navigateTo('/checklist/manage')` |
| `/important-dates` | 重要日期 | 打开「添加重要日期」弹窗 | 页面注册 `openAddModal()` |
| `/period` | 经期追踪 | 打开「记录经期」弹窗 | 页面注册 `openAddModal()` |
| `/vault` | 密码本 | 打开「新建条目」弹窗 | 页面注册 `openAddEntry()` |
| 其他页面 | 更多 / 商店 / 设置等 | 打开快速添加弹窗（默认） | Layout 默认行为 |

---

## 三、技术实现

### 3.1 Composable：`useGlobalAdd`

```ts
// composables/useGlobalAdd.ts
// 基于 provide/inject 的全局「+」按钮上下文注册机制
```

- **Layout（Provider）**：调用 `provideGlobalAdd()`，提供注册/注销接口
- **Page（Consumer）**：调用 `useGlobalAdd(handler)` 注册页面级处理函数
- 路由切换时自动清理注册，避免残留

### 3.2 工作流程

1. `layouts/default.vue` 调用 `provideGlobalAdd()` 初始化
2. 页面组件 `onMounted` 时调用 `useGlobalAdd(() => { ... })` 注册自定义处理
3. 用户点击「+」按钮 → Layout 检查是否有页面注册的 handler
   - **有** → 执行页面 handler
   - **无** → 执行默认快速添加弹窗逻辑
4. 页面 `onUnmounted` 时自动注销 handler

### 3.3 注意事项

- 每次只能注册一个 handler（后注册覆盖先注册）
- 路由守卫在 `beforeEach` 中清理，确保跳转后不残留
- 页面无需关心 Layout 的默认逻辑，只需注册自己的行为
